¶ ? Wi-Fi

Os equipamentos VIRLOC 8 (VL08) e VIRCOM 7 (VC7) possuem conectividade Wi-Fi, permitindo:

  • Envio de reportes e caixa-preta para o servidor;
  • Cadastro de até 8 redes Wi-Fi diferentes;
  • Vale lembrar que o VIRLOC irá sempre priorizar o envio de pacotes utilizando a rede wireless, caso ele consiga se conectar em alguma.

 

¶ ⚙️ Configuração

É possível configurar até 8 redes distintas para que o Virloc tenha conectividade via Wi-Fi com o comando:

>VSWI20,"ssid""pass",0000<

Resposta:

>VRWI20,"ssid""pass",0000,50,102,-49<

Onde: 

VSWI20 = Índice da rede (20 a 27)

ssid = SSID da rede;

pass = Senha da rede;

0000 = Retardo de conexão de 0000 segundos;

50 = Estado de conexão (0 = desconectado, 20 = esperando conexão, 50 = conectado);

102 = Presente por 102 segundos (este contador se atualiza em 120s quando recebe a lista de APs e esse AP está presente);

-49 = Nível de sinal.

Para consultar a qualquer momento, basta enviar >VQWI20<

 

¶ ? Consulta de Conexão

Para consultar o estado da conexão, pode ser enviada a consulta:

>VQWF_ST<

Reposta:

>VRWF_ST,10,SU:100000000,SC:100000000,JAMhome,d8:30:62:30:25:7f,192,168,1,19,255,255,254,0,192,168,1,1,LE:0<

Onde:

10 = Link establecido

SU:10000000 = Sockets Ativados

 SC:10000000 = Sockets conectados ao servidor

JAMhome = Nome do AP ao qual está conectado

d8:30:62:30:25:7f = MAC Address do AP

192,168,1,19 = IP local

255,255,254,0 = Máscara subnet

192,168,1,1 = IP gateway

LE:0 = Link Error

 

¶ Envio de reportes via Wi-Fi

É possível programar o VIRLOC para enviar reportes utilizando a conectividade Wi-Fi. Para isso, utilizamos WFx como destino, sendo x o índice do socket do servidor.

 

¶ ? Exemplos de eventos

? Evento que envia reporte para o destino GF0, podendo ser enviado via Wi-Fi, GPRS ou LoRa (conforme disponibilidade das conexões).

>SED000 IN07++ +- GF0 V1<

 

? Evento que envia reporte para o destino WF0, podendo ser enviado somente via Wi-Fi.

>SED001 IN07++ +- WF0 V1<

 

¶ Reporte de conexão

Quando o Virloc se conecta a uma rede Wi-Fi, ele envia automaticamente um reporte RUV para abrir a conexão com o servidor.

Por padrão, o reporte enviado será:

>RUV99999,34L10099,cc:93:4a:19:a3:56<

Onde:

34L10099 = Serial do dispositivo

cc:93:4a:19:a3:56 = MAC Address do dispositivo

 

¶ ?️ Reporte configurável ao conectar

Você pode configurar o Virloc para enviar um reporte configurável em vez do RUV padrão:

Também é possível enviar um reporte configurável através da configuração:

>VSWF4,215,30,5,0,0,x<

Onde:

 x =  o índice do reporte configurável a ser enviado.

 

¶ ? Disparador de Conexão Socket (0 a 7)

Disparadores WFxx++ (30 a 37) a conexão com os sockets de servidor VSIPx (0 a 7).

 WF30++ = Dispara quando conectado ao socket 0;

 WF30-- = Dispara quando desconectado ao socket 0.

 

? Exemplos de uso:

>SED00 WF30++ +- GF0 V1< = Dispara ao conectar no socket 0.

>SED01 WF30-- +- GF0 V1< = Dispara ao desconectar do socket 0.

 

¶ ? Reinicio do módulo Wi-Fi

É possível programar o Virloc para reiniciar o modulo WI-FI, caso não conseguir conectar em nenhuma das redes cadastradas no equipamento, para isso utilizado o comando a seguir:

 

¶ ? Habilitar reinício automático

>VSWF4,215,30,5,0,0,99,A5<

 

¶ ? Desabilitar reinício automático

>VSWF4,215,30,5,0,0,99,A6<

 

¶ ? Automatismo de Baud Rate

O automatismo de baud rate em Wi-Fi se refere a um mecanismo de autoajuste da taxa de transmissão de dados, dependendo das condições do canal e da qualidade do sinal. 

No contexto de redes Wi-Fi ou dispositivos IoT, esse automatismo pode ser implementado para otimizar a comunicação.

Esse função está habilitada a partir versão de FW 158B20. 

>SRS4 115200,0D,115200,460800<

¶ ? Bluetooth

Os equipamentos VIRLOC 8 e VIRCOM 7 possuem conectividade Bluetooth, permitindo:

  • Comunicação com o computador (configuração via terminal/app);
  • Integração com aplicativos em tablets e smartphones;
  • Envio de reportes usando o destino BTT.

 

¶ ? Pareamento

  1. No computador, smartphone ou tablet, busque por dispositivos Bluetooth próximos;
  2. Localize o Virloc pela identificação de nome (ver abaixo);
  3. Selecione e pareie – não é necessário PIN.

 

 

¶ ✏️ Alterar nome do dispositivo Bluetooth

Por padrão, o nome do dispositivo será:

VIRTEC_VL8_XXXX

Onde:

  • XXXX = é o ID do dispositivo.

 

Para alterar o nome, pode ser utilizado o comando abaixo:

>VSWI16,AT+SRBTNAME="VIRTEC_VL8_%ID"<

Onde:

VIRTEC_VL8_ = String personalizável;

%ID = Referência ao ID configurado no dispositivo. Outro valor possível é %SN referenciando o Serial do dispositivo.

⚠️ Atenção:
Ao alterar o nome do dispositivo, o módulo será reiniciado automaticamente.

 

¶ ? Envio de reportes para destino Bluetooth

É possível programar o VIRLOC para enviar reportes através do Bluetooth. Para isso, utilizamos BTT como destino.

 

Exemplos:

  • Prioridade NORMAL: Evento enviará o reporte para o destino BTT realizando até 5 tentativas de envio, esperando ACK para considerar o reporte enviado com sucesso.

>SED000 IN07++ +- BTT V1<

 

  • Prioridade BAIXA: Evento enviará o reporte para o destino BTT realizando apenas 1 tentativa de envio.

>SED001 IN07++ +- BTT_LO V1<

 

  • Prioridade ALTA: Evento enviará o reporte para o destino BTT realizando tentativas até receber ACK.

>SED002 IN07++ +- BTT_HI V1<

¶ ? Controle de energia e reinicialização

A gestão de alimentação do Bluetooth e Wi-Fi nos Virlocs é controlada simultaneamente com os comandos abaixo:

 

¶ ? Desligar Wi-Fi e Bluetooth

>VSLP_WF0<

 

¶ ? Ligar Wi-Fi e Bluetooth

>VSLP_WF1<

 

Também é possível reiniciar ou restaurar as configurações utilizando os comandos:

 

¶ ? Reinicializar o módulo (sem perder configurações)

>VSWI_INI<

 

¶ ? Restaurar strings de configuração e reiniciar

>VSWI_DEF<

 

¶ ? Restaurar módulo para configurações default e reiniciar

>VSWI_RST<

 

¶ ℹ️ Informações do módulo

É possível consultar informações do módulo Wi-Fi e Bluetooth por meio da consulta:

>VQWF<

 

Retorno:

>VRWF4,215,30,5,0,0,99,X2205600951310,cc:93:4a:20:d2:9a,BX310x.2.7.3,000;ID=0061;#80BE;*4A<

Onde:

X2205600951310 = Número de série

cc:93:4a:20:d2:9a = MAC Address

BX310x.2.7.3 = Versão de firmware

 

¶ ?️ Versões de hardware

É possível identificar a versão de Bluetooth verificando a etiqueta do seu Virloc.

  • Placas contendo a identificação BX possuem o módulo anterior com Bluetooth Clássico.
  • Placas contendo a identificação FC possuem o módulo novo com BLE.

 

Também é possível identificar através da consulta VQWF:

  • Módulo anterior (BX3105) com Bluetooth Clássico:

>VRWF4,215,30,5,0,0,99,X2205600651610,cc:93:4a:21:2e:86,BX310x.2.7.2,000,0;ID=0069;#8011;*04<

  • Módulo novo (FC41D) com BLE:

>VRWF4,215,30,5,0,0,99,,e8:47:27:bf:dd:ae,FC41D_VERSION:FC41DAAR03A02.bin,000,1;ID=0000;#80A4;*60<

 

¶ 🆕 Wi-Fi - Novidades de firmware acima de 156

¶ 🕶️ Cadastro de redes ocultas (FW 156B03)

A partir do firmware 156B03 o comando VSWI20 aceita um parametro adicional para indicar que a rede e oculta (SSID nao publicado). O equipamento conecta usando o SSID e senha mesmo sem broadcast.

Exemplo (rede oculta):

>VSWI20,"ssid_oculto","senha",0001<

O ultimo parametro (0001 em vez de 0000) marca como rede oculta.

¶ 📡 WI08 - Datatype FC41 (FW 156B05)

A partir do firmware 156B05, no modulo WiFi FC41, o campo de datatype foi atualizado para suportar mensagens com tamanhos diferenciados no comando VSWI08.

¶ 📨 Mensagens Wi-Fi > 128 caracteres (FW 156B17)

A partir do firmware 156B17 mensagens enviadas/recebidas via Wi-Fi suportam mais de 128 caracteres em um mesmo pacote.

¶ 🔄 Reset do modulo WiFi - VSWF4 A5/A6 (FW 156B38)

A partir do firmware 156B38 e possivel realizar reset suave (A5) ou reset hard (A6) do modulo WiFi sem reiniciar o equipamento todo:

>VSWF4,A5<

Reset suave (reinicia apenas a stack TCP/IP).

>VSWF4,A6<

Reset hard (power cycle do modulo).

¶ 📶 Disparadores Wi-Fi (FW 156B07 / 157B13 / 158B21)

Novos disparadores adicionados:

  • WF20++ (FW 156B07) - equipamento entrou online via Wi-Fi (recebeu IP)
  • WF20-- (FW 156B07) - perdeu conexao Wi-Fi
  • WF30++ a WF37++ (FW 157B13) - sockets 0 a 7 conectaram individualmente
  • WF30-- a WF37-- (FW 158B21) - sockets 0 a 7 desconectaram individualmente

¶ 📛 VSWIxx offset IP (FW 160B25)

A partir do firmware 160B25 e possivel adicionar o sufixo #x (onde x = 0-F) no comando VSWIxx para apontar a rede Wi-Fi a um IP de destino especifico (offset). Util quando o equipamento possui multiplos destinos servidores.

¶ 💾 Entrega de LEX via Wi-Fi (FW 154P01)

A partir do firmware 154P01 a entrega da caixa preta (LEX) pode ser feita via Wi-Fi quando disponivel, reduzindo custo com GSM/4G para dispositivos com conectividade local.

¶ 🆕 Bluetooth - Novidades de firmware acima de 156

¶ 🔗 Disparadores de conexao BT (FW 159B16 - modulo FC41)

Para equipamentos com modulo Bluetooth FC41, a partir do firmware 159B16, disparadores BT00++/-- e BT01++/-- acionam quando um dispositivo BT Classic ou BLE conecta/desconecta.

¶ 🔌 Multiconexao Bluetooth (FW 154B12)

A partir do firmware 154B12 o modulo Bluetooth permite multiplas conexoes simultaneas com perifericos diferentes (ex.: leitor RFID + iButton + celular).

¶ 📝 BT FC41 - Caracteristicas estendidas (FW 156B17 / 156B27)

A partir do firmware 156B17 o comando AT+QBLEGATTSNTFY do FC41 aceita 0xFF01 como caracteristica. Em 156B27 o BLE passou a aceitar caracteristicas com letras minusculas no UUID.

¶ 🔍 VQWF - Consulta tipo de modulo (FW 155B22 / 158B21)

A consulta VQWF (FW 155B22+) retorna o tipo de modulo BT/WiFi presente no equipamento (modelo, firmware). Em 158B21 foi adicionado VQWF_ST que retorna estado complementar do WiFi.

¶ 📢 GFx via Bluetooth Classic/TRM (FW 158B21)

A partir do firmware 158B21 os destinos de envio GFx podem ser direcionados para Bluetooth Classic ou para um periferico via TRM (terminal/serial transparente).

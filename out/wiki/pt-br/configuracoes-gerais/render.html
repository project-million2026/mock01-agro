¶ Configuração de energia Periféricos VSPK

A configuração de VSPK define os parâmetros e limites de tensão para ligar e desligar periféricos, carregamento da bateria interna e parâmetros para detecção da ignição.

 

>VSPK,2350,2200,1170,1030,0360,0340,0360,0340,0350,0340,0430,0420,0340,0800,0450,+50,+70,0000,0000,0000,999,A5,001<

Onde: 

2350 = Tensão para detectar alimentação principal de 24V presente (mínimo: 15V [1500])

2200 = Tensão para detectar a ausência de alimentação principal de 24V (mínimo: 15V [1500])

1170 = Tensão para detectar alimentação principal de 12V presente

1030 = Tensão para detectar a falta de alimentação principal de 12V

0360 = Tensão da bateria para ligar o GPS e os periféricos

0340 = Tensão da bateria para desligar os periféricos GPS 

0360 = Tensão da bateria para ligar o MDM

0340 = Tensão da bateria para desligar o MDM

0350 = Tensão da bateria para ligar EVP (eventos)

0340 = Tensão da bateria para desligar o EVP (eventos)

0430 = Limite de tensão de carga da bateria interna

0425 = Tensão de carregamento para tensão constante da bateria interna

0340 = Tensão da bateria interna para carregamento de corrente constante

0800 = Corrente da carga rápida

0450 = Corrente da carga residual

+50 = Temperatura interna para limitar a corrente de carga da bateria (em corrente de carga rápida) [+25…+69, padrão = +50]

+70   = Temperatura interna para limitar a tensão do carregador ao mínimo para não carregar a bateria [+25…+69, padrão = +60]

0000= Tensão para detecção IGNIÇÃO VIRTUAL para 12V. Por padrão, está desabilitada em 0000

0000 = Tensão de detecção IGNIÇÃO VIRTUAL para 24V. Por padrão, está desabilitada em 0000

0000 = Histerese para detecção de IGNIÇÃO VIRTUAL

999  = Tensão para acordar (sleep) por bateria carregada (utilizado somente para equipamentos que possuem carregamento de baterias externo), está desabilitado em 9999

A5   = Define o tipo de ignição 

  • ignição física por entrada analógica = A5
  • ignição virtual por valor de CT/RPM = A6
  • ignição virtual por valor de tensão de alimentação = FF

001 = Define fonte da ignição (quando definido A5 ou A6 no parâmetro anterior)

  • para ignição física por entrada analógica (A5 no parâmetro anterior) seleciona a entrada analógica (001 para AD1 - entrada IN00 / fio marrom)
  • para ignição virtual por valor de CT/RPM (A6 no parâmetro anterior) seleciona o índice do CT para valor de referência

 

⚠️ O parâmetro de histerese para detecção de ignição, define a diferença (delta de tensão) para reconhecimento de ignição ligada ou desligada. 
Assim este delta deve ser somado ao valor definido em Tensão de detecção IGNIÇÃO VIRTUAL para 21 ou 24V. 

? Exemplo:

>VSPK,2150,2000,1100,1070,0360,0340,0350,0340,0350,0340,0430,0425,0340,0800,0450,+50,+70,1300,2650,0020,999,FF,255<

 

  • Em 12V: 

1300 =  define a tensão de detecção IGNIÇÃO VIRTUAL com 12,00 Volts

0020 = define a tensão de 0,2 Volts para histerese 

Assim: 

  • Reconhece IGNIÇÃO LIGADA ao subir acima de 13,00 Volts
  • Reconhece IGNIÇÃO DESLIGADA ao cair abaixo de 12,8 Volts (13V - 0,2V = 12,8V)

 

  • Em 24V

2650 =  define a tensão de detecção IGNIÇÃO VIRTUAL com 24,00 Volts

0020 = define a tensão de 0,2 Volts para histerese 

Assim: 

  • Reconhece IGNIÇÃO LIGADA ao subir acima de 26,50 Volts
  • Reconhece IGNIÇÃO DESLIGADA ao cair abaixo de 26,3 Volts (26,5V - 0,2V = 26,3V)

 

A utilização de ignição virtual por tensão ainda correlaciona os fatores de velocidade do veículo e detecção de movimento (acelerômetro). Para configurar esses fatores, utilizar os parâmetros abaixo:

 

  • Nível de aceleração medida nos eixos X ou Y para considerar ignição ligada:

>VS16,3E,100<

Onde:

 100 = Nível na escala 1G = 100.

 

  • Velocidade mínima do GPS para considerar ignição ligada:

>VS16,40,30< 

Velocidade do GPS para forçar ignição se for definida por tensão

Onde:

 30 = Velocidade em km/h.

 

  • Tempo para manter a ignição ligada ao detectar movimento por velocidade ou acelerômetro:

>VS16,42,600<

Onde:

 600 = Tempo em décimos de segundo (600 = 60s).

 

¶ Calibração das Entradas Analógicas

Para calibrar uma entrada analógica, conecta-se uma fonte de tensão ao VIRLOC através da entrada desejada e utiliza-se o seguinte comando:

 

>SOMpppp,XX,10000,10000<

Onde:

pppp = Senha geral do equipamento

XX = Número da Entrada analógica

10000 = Fator de calibração

Recomenda-se a utilização de um equipamento de medição de tensão como multímetro ou voltímetro uma vez que a tensão da fonte conectada na entrada analógica é desconhecida.

 

Consulta da tensão na entrada analógica:

>QOMxx<

Onde:

xx = Número da Entrada analógica

 

Retorno: 

>ROM01,10000,09780 1335;ID=0859;#8233;*6D< 

Onde:

01 = Entrada analógica 01

1359 = Tensão lida

 

¶ Ativar/Desativar Pull-up de Entrada

Nas entradas digitais é possível ativar ou desativar o pull-up individualmente para cada entrada.

⚠️ Exceção: a entrada 0 (fio marrom) não permite configuração de pull-up.

 

>SITx_PUP0<

Onde:

x = Entrada

0 = Desativado ou 1 = Ativado

 

¶ Configurações leitura ignição 

 

¶ Ignição Física (VIRLOC 08 e VCONE 108)

No Virloc 08 e no VCONE 108, a entrada IN0 (fio marrom) pode ter como uma das funções a entrada para reconhecimento de ignição física.

Para isso devem ser configurado os parâmetros:

 

  1. Definir a entrada 0 (IN0) como entrada analógica 01 (QAD01 / RAD01):

  >SIT0003003_AD<

 

 2. Configurar o reconhecimento de ignição no VSPK:

 >VSPK,2150,2000,1100,1070,0360,0340,0350,0340,0350,0340,0430,0425,0340,0800,0450,+50,+70,1300,2500,0200,999,A5,001<

Onde: 

1300 = Tensão [Volts x 10] (0000 - 3000 Voltsx100 | 0 - 30 Volts) de detecção IGNIÇÃO em IN0 quando operando na faixa de 12V (na alimentação principal)

2500 = Tensão [Volts x 10] (0000 - 3000 Voltsx100 | 0 - 30 Volts) de detecção IGNIÇÃO em IN0 quando operando na faixa de 24V (na alimentação principal)

0200 = histerese para detecção de IGNIÇÃO ( 2,0 Volts)

A5 = Define o tipo da ignição. ignição física por entrada analógica = A5

0001 = Define fonte da ignição: para ignição física por entrada analógica (A5 no parâmetro anterior) seleciona a entrada analógica (001 para AD1 - entrada IN00 / fio marrom). 

Assim: 

  • Reconhece IGNIÇÃO LIGADA ao ler tensão acima da 13,0 Volts (1300) em IN00
  • Reconhece IGNIÇÃO DESLIGADA ao ler tensão abaixo de 11,0 Volts (1300 - 0200 = 1100) em IN00

 

¶ Ignição virtual por CT/RPM

A leitura da ignição pode ser definida também por CT, assim a referência será a leitura do valor gravado em um contador CT, podendo este armazenar o RPM do motor para definir a condição de ignição ligada.

Configurar o reconhecimento de ignição VSPK:

 >VSPK,2150,2000,1100,1070,0360,0340,0350,0340,0350,0340,0430,0425,0340,0800,0450,+50,+70,0100,0100,0020,999,A6,090<

Onde: 

0100 = limite reconhecimento ignição: define IGNIÇÃO LIGADA quando o valor do CT90 for maior ou igual a 100, quando operando na faixa de 12V (na alimentação principal)

0100 = limite reconhecimento ignição: define IGNIÇÃO LIGADA quando o valor do CT90 for maior ou igual a 100, quando operando na faixa de 24V (na alimentação principal) 

0020 = Histerese para detecção de IGNIÇÃO: define IGNIÇÃO DESLIGADA quando o valor do CT90 for menor que 80 ( 100 - 20 = 80)

A6 = Define o tipo de ignição: ignição virtual por valor de CT/RPM = A6

090 = seleciona o índice do CT para valor de referência CT90

¶ Configurações do Modem

Para utilizar a comunicação GSM é necessário configurar os dados de APN, usuário e senha de cada SIM card a ser utilizado, sendo que:

  • VIRCOM 07 disponibiliza 1 slot;
  • VIRLOC 08 disponibiliza 2 slots;
  • VIRLOC 12 possui opção single ou dual chip.

No VL08, o slot superior é o primário e o inferior é o secundário. Isso significa que o VL08 realizará o escaneamento primeiramente pelo slot superior e depois pelo inferior, assim, é recomendado utilizar o slot superior se for utilizar apenas um SIM Card para obter maior velocidade em casos de reconexão.

Ao configurar a APN com os comandos indicados a seguir, também serão criados os eventos ED50 a ED58 para gestão da comunicação, e o TD01.

 

¶ Configuração de APN do slot primário/único

>SCF0000,MX2,APN zap.vivo.com.br,SUP2 vivo vivo<

Onde:

0000 = Senha do equipamento

zap.vivo.com.br = APN do SIM1

vivo= Usuário do SIM1

vivo= Senha do SIM1

 

Consulta da APN configurada:

>QISF<

Resposta:

>RISF"zap.vivo.com.br"r;ID=LN8T;#8BF4;*3E<

 

Consulta de usuário e senha configurados:

>QUP2<

Resposta:

>RUP2 vivo vivo;ID=LN8T;#8BF5;*14<

 

¶ Configuração de APN para dual SIM

>SCF0000,MX2,APN timbrasil.br,SUP2 tim tim, APN2 generica.claro.com.br<

>SUP3 claro claro<

Onde:

0000 = Senha do equipamento

timbrasil.br = APN do SIM1

tim = Usuário do SIM1

tim= Senha do SIM1

generica.claro.com.br = APN do SIM2

claro= Usuário do SIM2

claro= Senha do SIM2

 

Consulta da APN configurada (slot 1):

>QISF<

Resposta:

>RISF"timbrasil.br"r;ID=LN8T;#8BF1;**40<

 

Consulta de usuário e senha configurados (slot 1):

>QUP2<

Resposta:

>RUP2 tim tim;ID=LN8T;#8BF1;*10<

 

Consulta da APN configurada (slot 2):

>QISf<

Resposta:

>RISf"generica.claro.com.br"r;ID=LN8T;#8BF0;*14<

 

Consulta de usuário e senha configurados (slot 2):

>QUP3<

Resposta:

>RUP3 claro claro;ID=LN8T;#8BF2;*12<

 

¶ Configurações do protocolo GPRS

>SGRxxxxyyzzPtttt, aabbcc, ddeeff, gghhii, jj, kk MUX MCmmmm Mxx<

Onde:

xxxx = Base de tempo entre as tentativas de envio (0000 ..9999 = 000.0 segundos .. 999,9 segundos);

yy = Número de tentativas de enviar mensagens de prioridade NORMAL;

zz = Número de tentativas de envio de mensagens de ALTA prioridade;

P = Número da porta para este protocolo (1 para COM1 ou 2 para COM2);

tttt = Tempo para reiniciar o modem (0000 a 9999 segundos);

aa = Número de segundos entre as tentativas do LCP;

bb = Número de tentativas na configuração do LCP;

cc = Número de tentativas de conclusão do LCP;

dd = Número de segundos entre as tentativas de PAP;

ee = Número de tentativas de configuração do PAP;

ff = Número de tentativas de conclusão do PAP;

gg = Número de segundos entre tentativas de IPCP;

hh = Número de tentativas de configuração IPCP;

ii = Número de tentativas de conclusão do IPCP;

jj = Período de tempo entre qualquer pedido de attach;

kk = Quantidade mínima de sinal para tentar entrar online;

MUX (opcional) = Este parâmetro deve ser definido se você quiser trabalhar com o modo MUX;

mmmm = tempo mínimo de conexão;

xx = Multiplicador atual do tempo entre tentativas (este parâmetro é para consulta).

 

OBS: O dispositivo irá reiniciar o MODEM a cada tttt segundos quando estiver no modo GPRS ON LINE.

 

¶ Reset do modem e troca manual de SIM

Para reiniciar o modem GPRS, utiliza-se o comando:

>SVP0 2<

No mesmo comando, é possível realizar a troca manual de SIM card utilizado (válido apenas para VL08 e VL12 dual SIM). Os Virlocs fazem esse chaveamento automaticamente, conforme a disponibilidade de rede.

 

>SVP0 2 SIM0< → Reinicia o modem e força o uso do SIM0 (slot superior)

>SVP0 2 SIM1< → Reinicia o modem e força o uso do SIM1 (slot inferior)

 

Ao consultar >QVP0 2< é possível verificar qual SIM card está sendo utilizado:

 

>RVP001 2 SIM0 NXSIM1 BAND1,1,2,2,5 FIX0;ID=0070;#8012;*21<

Onde:

SIM0 = Sim card sendo utilizado atualmente (0: slot superior | 1: slot inferior)

NXSIM1 = Slot que será utilizado após o próximo reset (0: slot superior | 1: slot inferior)

 

¶ Configurações do servidor de destino

Podem ser configurados até 16 sockets para servidor de destino. Cada um deles é referenciado na programação de eventos como GFx (sendo x o índice do socket).

? Exemplo:

>SED001 IN07++ GF0 GP<

 

¶ Configuração do servidor por IP

 

Configuração para servidor UDP:

>VSIPa,xxx.xxx.xxx.xxx.bbbbb.ccccc,UDP,AUXd,RF0,ETH0,CAN0,TRM0,BTT1,GSM0<

Onde:

a = índice do socket em hexadecimal [0 a F]

xxx.xxx.xxx.xxx = IP do servidor

bbbbb = porta de origem (padrão: 04096)

ccccc = porta de destino (padrão: 04097)

UDP = protocolo de comunicação UDP

d = índice do socket backup em hexadecimal [0 a F] - parâmetro opcional (padrão: AUXF)

RF = Habilita descarga via Lora

ETH = Habilita descarga via Ethernet

TRM = Habilita descarga via Serial

BTT = Habilita descarga via bluetooth

GSM= Habilita descarga via GSM

 

  • Agregado configuração GSM e BTT a VSIP para definir se esse socket sairá por GSM e por BT.
  • Sockets 8 a F sempre estão habilitados para sair por GSM e não permitem ser habilitados por BTT ou WFx.
  • Sockets 0 a 7 permitem ser configurados para enviar por BTT o WIFI.

 

Configuração para servidor TCP:

>VSIPa,xxx.xxx.xxx.xxx.bbbbb.ccccc,SC01,SECF7,TCP0768,0768,ST3,ACK,AUXd,RF1<

Onde:

a = índice do socket em hexadecimal [0 a F]

xxx.xxx.xxx.xxx = IP do servidor

bbbbb = porta de origem (padrão: 04096)

ccccc = porta de destino (padrão: 04097)

TCP0768 = protocolo de comunicação TCP

ACK = desabilita a exigência de ACK

d = índice do socket backup em hexadecimal [0 a F] - parâmetro opcional (padrão: AUXF)

 

Consulta:

>VQIPa<

 

Resposta:

>VRIPa,xxx.xxx.xxx.xxx.bbbbb.ccccc,SCxx,SECFF,UDP,AUXd<

Onde:

a = índice do socket em hexadecimal [0 a F]

xxx.xxx.xxx.xxx = IP do servidor

bbbbb = porta de origem (padrão: 04096)

ccccc = porta de destino (padrão: 04097)

SCxx = estado da conexão [00 = offline | 01 = online]

SECFF = mapa de bits

UDP = protocolo de comunicação [TCP ou UDP]  (padrão: UDP)

d = índice do socket backup em hexadecimal [0 a F] (padrão: AUXF)

 

Importante:

Quando estiver utilizando mais de um IP de destino, é necessário habilitar o incremento do ponteiro de entrega realizando uma alteração no evento de sistema ED202, conforme exemplificado abaixo.

  • Incremento do ponteiro de entrega do IP0 (padrão): >SED202 TT00++ +- SGN GP {GGFN0}<
  • Incremento do ponteiro de entrega dos IPs 0 e 1: >SED202 TT00++ +- SGN GP {GGFN0,1}<
  • Incremento do ponteiro de entrega dos IPs 0, 1 e 14: >SED202 TT00++ +- SGN GP {GGFN0,1,E}<

 

  • Agregado opcional a TCPclient para usar porta aleatória

O bit 3 de SECxx define se será utilizada uma porta aleatória ou fixa definida em VSIP

  Ex.

SECF8 = Define porta fixa

 SECF0 = Define porta aleatória

  • TCP ativa portas físicas habilitadas para este IP. Quantidade máxima de conexões alterada para 4. Em firmwares anteriores reservava uma porta para LAN e outra para GSM

Ex.

>VSIP6,186.122.150.153.8046.8047,TCP768,768,RF0,ETH1,CAN0,TRM0,BTT0,GSM0<

 Só ativa TCP para LAN e não para GSM

 

¶ Configuração do servidor por DNS

Para utilizar a configuração por DNS, deve ser enviado o comando VSIP com as configurações do socket conforme descrito acima, porém com o IP 1.1.1.1

Em seguida, deve ser enviado o comando VSDN com o DNS a ser utilizado.

 

Configuração:

>VSIPa,1.1.1.1.bbbbb.ccccc,UDP,AUXd<

>VSDNa,dominio.com<

Onde:

a = índice do socket IP associado

1.1.1.1 = IP do socket (deve obrigatoriamente ser configurado dessa forma)

bbbbb = porta de origem (padrão: 04096)

ccccc = porta de destino (padrão: 04097)

UDP = protocolo de comunicação [TCP ou UDP] - parâmetro opcional (padrão: UDP)

d = índice do socket backup em hexadecimal [0 a F] - parâmetro opcional (padrão: AUXF)

dominio.com = domínio do destino para o socket a

 

Consulta:

>VQDNa<

 

Resposta:

>VRDNa,dominio.com<

Onde:

a = índice do socket IP associado

dominio.com = domínio do destino para o socket a

 

¶ Alocação de memória para Buffers, LOGs e Regiões

A divisão e configuração da memória reservada para Buffer FIFO/LIFO (GFx | WIFI | Lora-Mesh), LOG1, LOG2 e Log Estendido LEX pode ser configurada pelo comando VSLG.

A memória total dos Virlocs 08, 12 e Vircom 07 é de 128Mb, o que permite a alocação de 900.000 de registros.

Essa memória é compartilhada entre os buffers, LOGs e Regiões, portanto aumentar o espaço reservado para um deles, reduz o espaço livre para os demais.

 

Configuração:

>VSLG0000,1024,8192,0,FOTA,0,LEX8192,FF8192<

Onde:

0000 = Senha geral do equipamento (padrão: 0000);

1024 = Quantidade de reports para Buffer FIFO/LIFO (GFx | WIFI | Lora-Mesh); Valores permitidos: 0, 512, 1024, 2048, 4096, 8192, 16384 e 32768;

8192 = Quantidade de reports para LOG1. Pode ser configurado com valores múltiplos de 256 (mínimo: 8192);

0 = Quantidade de reports para LOG2, não é um parâmetro configurável (aparecerá apenas na consulta VQLG), indica o restante de memória após a configuração de LOG1.

FOTA = Reservar memória ou não para atualizações de firmware OTA [FOTA = habilita | NOFOTA = desabilita];

0 = Habilita backup de reportes descartados pelo buffer FIFO/LIFO. Pode ser configurado como LOG (para backup em LOG1), LG2 (para backup em LOG2), FXT (para backup em SD card) ou 0 para desabilitar;

LEX8192 = Quantidade de reports para Log Estendido LEX. Pode ser configurado com valores múltiplos de 256;

FF8192 = Quantidade de reports para RF_FIFO (buffer de mensagens para comunicação entre os dispositivos na rede lora-mesh). Pode ser configurado com valores múltiplos de 256.

 

Consulta:

>VQLG<

 

Retorno:

>VRLG____,1024,00008192,00385024,1(FOTA)__,0(GFX_NO_BAK),8192,8192<

Onde:

0000 = Senha geral do equipamento (padrão: 0000);

1024 = Quantidade de reports para Buffer FIFO/LIFO (GFx | WIFI | Lora-Mesh); Valores permitidos: 0, 512, 1024, 2048, 4096, 8192, 16384 e 32768;

00008192 = Quantidade de reports para LOG1;

00385024 = Quantidade de reports para LOG2, não é um parâmetro configurável (aparecerá apenas na consulta VQLG), indica o restante de memória após a configuração de LOG1.

1(FOTA)__ = Memória para atualizações de firmware OTA [FOTA = habilita | NOFOTA = desabilita];

0(GFX_NO_BAK) = Backup de reportes descartados pelo buffer FIFO/LIFO. Pode ser configurado como LOG (para backup em LOG1), LG2 (para backup em LOG2), FXT (para backup em SD card) ou 0 para desabilitar;

8192 = Quantidade de reports para Log Estendido LEX. Pode ser configurado com valores múltiplos de 256;

8192 = Quantidade de reports para RF_FIFO (buffer de mensagens para comunicação entre os dispositivos na rede lora-mesh). Pode ser configurado com valores múltiplos de 256.

 

Importante:

  • A configuração VSLG deve ser enviada sempre ANTES de qualquer definição de região (VSRM), pois ela realoca a memória do Virloc e por isso pode corromper arquivos de áudio, strings e pontos de cerca embarcados.

 

¶ Definição de tipo de buffer (FIFO/LIFO)

Por padrão, o tipo de fila dos reportes nos equipamentos é FIFO (First in, First out, reportes mais antigos são enviados primeiro), para alterar a fila para LIFO (Last in, First out, reportes mais recentes são enviados primeiro), o evento 202 deve ser alterado, conforme abaixo:

>SED202 TT00++ +- SGN GP {GGFN0L}< → Define buffer GF0 para LIFO

>SED202 TT00++ +- SGN GP {GGFN0}< → Define buffer GF0 para FIFO (configuração padrão)

 

¶ Controle de Energia

Os periféricos integrados ao VIRLOC podem ser desligados e ligados utilizando o comando VSLP.

 

Configuração:

>VSLP_AAAx<

Onde:

AAA = periférico a ser manipulado, podendo ser:

  • GPS = módulo de GPS
  • MDM = modem GSM
  • INS = pullup das entradas digitais
  • VO1 = tensão Virnet
  • CHG = fonte da alimentação principal
  • SDC = módulo SD card
  • CAN = módulo CAN Bus
  • AUX = módulo leitor QR
  • MF = módulo leitor RFID
  • RF = módulo Lora
  • WF = módulo Wi-fi e Bluetooth
  • BCH = carregado da bateria interna

x = 1 = ligar periférico ou 0 = desligar periférico

 

Os parâmetros podem ser encadeados para ligar ou desligar periféricos com um único comando, por exemplo, para ligar o modem e desligar a comunicação Lora:

>VSLP_MDM1_RF0<

 

Consulta:

>VQLP<

 

Resposta:

>VRLP_GPS1_MDM1_INS1_VO11_CHG1_SDC1_CAN1_CLK64 LPev00 WUev00 WU1ev00000000 WU0ev00000000_ACLK64_AUX1_MF0_BCH1_RF1_WF1<

 

¶ Parâmetros do GPS

Esta configuração permite definir parâmetros mínimos de qualidade de sinal de recepção GPS para que as posições obtidas sejam inseridas nas consultas GP e TT, de forma a filtrar posições não aceitáveis.

 

Configuração:

>SGGaabbccddddefg<

Onde:

aa = filtro de HDOP para tomar uma posição como válida

bb = valor de HDOP a partir do qual se ativa o contador de tempo para reiniciar o módulo GPS

cc = mínimo número de satélites para evitar o reset

dddd = quantidade de tempo em segundos que esperará para reiniciar o módulo GPS

e = habilitar o reset por HDOP maior que bb

f = habilitar o reset por não posicionar com mais de cc satélites

g = habilitar o reset de GPS por ligar sem bateria interna de backup

 

Consulta:

>QGG<

¶ Antiestrelamento

A partir da versão de firmware 154B16 foi adicionado um parâmetro ao SGG para habilitar o sistema de assistência do acelerômetro para evitar estrelamento do GPS. Estrelamento basicamente acontece quando a posição do veículo é continuamente atualizada sem movimento; devido a precisão do GPS, a posição pode variar no decorrer do dia então se ela for atualizada mais de uma vez sem movimento, haverá um pequeno deslocamento no mapa mesmo que o veículo esteja parado.

O sistema de antiestrelamento fará com que a posição só seja atualizada quando houver movimento efetivo do veículo, evitando esse efeito.

 

Configuração:

>SGG_MVttttt,xx,yy,zz,sss<

 

Consulta:

>QGG_MV<

 

Resposta:

>RGG_MVttttt,xx,yy,zz,sssttttt<

Onde:

ttttt = Tempo para manter a posição quando nenhum movimento mais for detectado [0..65535 segundos];

xx = Nível de variação no eixo X para detectar o movimento [0..99 = 0..0,99 G];

yy = Nível de variação no eixo Y para detectar o movimento [0..99 = 0..0,99 G];

zz = Nível de variação no eixo Z para detectar movimento [0..99 = 0..0,99 G];

sss = Velocidade mínima para manter o movimento se já veio com movimento e velocidade [km/h] (Isso mantém a posição se houver movimento sem aceleração, mas com velocidade);

ttttt = Valor atual do contador de segundos sem detectar movimento.


O sistema é desativado configurando 65535 segundos no parâmetro ttttt.

¶ Senha Geral

Existem comandos que exigem uma senha para serem executados, que é a senha geral do equipamento. Para o VIRCOM 7, VIRLOC 8 e VIRLOC 12, a senha padrão é 0000.

 

É possível alterar essa senha usando o comando:

>SPEppppnnnn<

Onde:

pppp = senha atual (padrão: 0000)

nnnn = nova senha

 

Se a senha for alterada com sucesso, o equipamento irá responder:

>RPEpppp<
 

Ou, se houver erro na informação da senha atual:

>RPE_ERROR<

O ID do dispositivo é uma cadeia de 4 caracteres alfanuméricos que compõe as mensagens enviadas e recebidas por ele, e que representa o equipamento permitindo sua identificação. De fábrica é configurado um ID que pode ser modificado pelo usuário através do comando:

Onde:

iiii = novo ID a ser configurado

Antes de programar um equipamento com um novo script, é importante executar a limpeza através de um reset para a configuração de fábrica com o comando CLN. 

Esse comando desativa todos os protocolos nas portas de comunicação, apaga todos os eventos e variáveis. Pode ser enviado somente localmente via cabo ou bluetooth, sendo impossível enviá-lo remotamente.

>CLNpppp<

Atenção: Ao habilitar o protocolo TXT, não será possível enviar o comando. 

Para isso, é necessário desativar o protocolo TXT utilizando o comando NO CARRIER (fora do formato XVM).

>CLNpppp_FULL<

>CLNpppp_CAN<

 

¶ Apagar somente os EVENTOS

>CLNpppp_EVT<

 

¶ Restaurar eventos de sistema (200 a 202)

>CLNpppp_SYS<

 

¶ Restaurar todas as configurações 

Mantendo IP0, DN0, APN0, USR0, PASS0, EV50..58. 

O objetivo é poder limpar o equipamento remotamente sem perder a conexão:
(Disponível a partir da versão de firmware 154B05)

>CLNpppp_SRV<
 

O parâmetro SRV ainda pode ser combinado às outras opções do comando CLN citadas anteriormente.

A configuração TCFG03 define o tipo de veículo para broadcast das informações na rede Lora.

>TCFG03,8193<

Onde:

Ao ler um cartão RFID, primeiro analisa-se o código da tabela de veículos, se não conseguir encontrar o código na tabela de veículos, dispara CR09++ e verifica, se não consegue encontra-la dispara CR10++. Se o cartão foi encontrado e há apenas pessoal embarcado em verde (com permissão ativa), dispara no CR11++, se existe alguém em vermelho, dispara no CR12++.

¶ Configuração Ethernet (VCONE)

O VCONE possui uma porta Ethernet, podendo obter conexão diretamente de uma porta LAN de um modem externo ou roteador através de um cabo RJ-45.

A configuração pode ser realizada conforme descrito abaixo:

Consulta: 

>VQET< 

Resposta: 

>VRET192.168.000.250,255.255.255.000,192.168.000.001,DHCP1,MAC7A304F303D77,SC0000000000000000,LS:0,SPD::10M,st:255,dl:3773,err:005,cl0<

Configuração:

>VSET192.168.000.250,255.255.255.000,192.168.000.001,DHCP1<

 

Onde: 

192.168.000.250 = Endereço IP local

255.255.255.000= Máscara de subrede

192.168.000.001= Endereço gateway

DHCP1= DHCP habilitado [0..1]

MAC7A304F303D77= MAC address local

SC0000000000000000 = Estado de conexão dos sockets

LS:0 = Link Status

SPD::10M= Velocidade de conexão LAN (10M ou 100M)

st:255= Status (200 = DHCP ready)

dl:3773= Delay para ligar o driver em unidades de 10 ms.

 

Se o cabo estiver conectado, cada tentativa de estabelecimento de Link será feita a cada 60 segundos.

O contador dl indica o tempo para a próxima tentativa de conexão, onde 100 representa 1 segundo.

Recomenda-se definir a velocidade para 10M para evitar possível interferência em alguns canais do módulo RF.

 

¶ 🆕 Configuracoes adicionadas em firmware acima de 156 (VL08/VL12/VC07)

Comandos de configuracao geral adicionados em versoes recentes de firmware. Consulte >QVR< para validar a versao instalada antes de usar uma feature.

¶ 🔄 CLN_SRV - Reset seguro mantendo conectividade (FW 154B05)

A partir do firmware 154B05 o comando CLN_SRV realiza reset completo das configuracoes mas mantem IP0, DN0 e APN0. Util para reset remoto seguro - o equipamento mantem conexao com o servidor para receber novas configuracoes.

Exemplo:

>CLN_SRV<

¶ 💾 SSO - Salvamento automatico de CTs em flash (FW 158B21)

A partir do firmware 158B21 o comando SSO habilita o salvamento automatico de todos os CTs (192 contadores) na memoria flash interna. Permite que os contadores sobrevivam a reinicializacoes/quedas de energia.

Exemplo:

>SSO<

¶ 🆔 TCFG_DNI - ID invalido e CRD/NAK_CRD (FW 159B17)

A partir do firmware 159B17 o parametro TCFG_DNI permite definir o comportamento quando um cartao RFID/iButton com ID invalido (nao cadastrado) e apresentado. Compativel com eventos CRD (cartao reconhecido) e NAK_CRD (cartao rejeitado) com variaveis customizaveis.

¶ 🌐 TCP por porta fisica - LAN ou GSM (FW 160B25)

A partir do firmware 160B25 e possivel escolher por qual porta fisica (LAN/Wi-Fi ou GSM/4G) cada destino TCP saira. Permite priorizar conexao local em vez de celular para reduzir custos.

¶ 📡 SVP0 BAND - Selecao de banda LTE/CatM1 (FW 153B27)

O comando SVP0 2 BANDx permite forcar bandas LTE/CatM1 especificas no modem ou usar modo automatico. Em FW 156B23+ tambem aplica ao modem EG915 (4G LTE Cat-1).

¶ 📍 GGP0,y - Media de posicoes GPS (FW 156B26)

A partir do firmware 156B26 o comando GGP0,y calcula a media de y posicoes consecutivas. Util para reduzir tremor em paradas longas e melhorar precisao em ambientes com sinal fraco.

Exemplo (media de 5 posicoes):

>GGP0,5<

¶ 🌡️ Temperatura de carga da bateria (FW 154B18 / 156B28)

Em FW 154B18 a temperatura maxima de carga da bateria foi limitada a 60 graus C. A partir do FW 156B28 o limite default passou para 50/55 graus (warning/critical), com possibilidade de ajustar ate 60/65 graus em ambientes especificos. Protege a bateria de ciclos termicos extremos.

¶ 🔄 Reset separado de ISs - Modem, BT, DNS, APN, WI (FW 155B17 / 154B05)

A partir do firmware 154B05 (com refinamento em 155B17) e possivel resetar cada bloco de configuracoes independentemente, em vez de fazer reset total:

  • CLN_MDM - Limpa apenas configuracoes do modem (APN, IPs)
  • CLN_BT - Limpa apenas configuracoes Bluetooth
  • CLN_DNS - Limpa apenas DNs cadastrados
  • CLN_APN - Limpa apenas APNs cadastradas
  • CLN_WI - Limpa apenas redes Wi-Fi cadastradas

¶ 📱 ICCID limpa apos reset do modem (FW 155B17)

A partir do firmware 155B17, apos reset do modem ou troca de SIM, o ICCID interno e limpo automaticamente e re-lido. Antes era necessario reiniciar o equipamento todo para que o ICCID fosse atualizado.

¶ 🔋 Verificacao de presenca de bateria (FW 154B23)

A partir do firmware 154B23 o sistema detecta ausencia de bateria interna e reporta atraves de GCH estado 6 (bateria ausente). Em FW 156B38 ainda foi adicionado o estado BT-999 que indica que o carregador esta desconectado.

¶ 🔑 VS16,2C - Tempo de manutencao apos ignicao desligar

O comando VS16,2C,xxx define em decimos de segundo o tempo durante o qual o equipamento mantem a ignicao como ATIVA mesmo apos o sinal IN07 cair. Util para evitar gera multiplos eventos de igniton off/on em veiculos com flutuacao de tensao.

Exemplo (manter ignicao 5 segundos apos desligar):

>VS16,2C,050<

¶ 🆔 VSIP - Prefixo de letra por destino (FW 157B13)

A partir do firmware 157B13 o comando VSIP aceita o parametro ID0x (x = 0-F) que altera a primeira letra dos reportes enviados para esse destino especifico. Permite que o servidor identifique qual destino enviou a mensagem.

Exemplo (destino 0 com letra T como primeira letra do ID):

>VSIP0,1.2.3.4,12345,0,F,ID0T<

¶ 🚗 VS08,07 - Fonte de velocidade do equipamento

O comando VS08,07,xx define a fonte de velocidade considerada pelo equipamento:

  • VS08,07,99 - Velocidade GPS (default)
  • VS08,07,xx - CTxx multiplicado por 1000 (ex.: xx=12 usa CT12 como velocidade em km/h)

Util para usar velocidade lida do CAN (RPM, pulsos, OBD2 PID 0D) como referencia para limiares de antiestrelamento, eventos de movimento e contadores estatisticos.

¶ 🚦 GS - Faixas reduzidas para 16 (FW 160B25)

A partir do firmware 160B25 o numero de faixas de velocidade GS (Group Speed) foi reduzido de 32 para 16 (indices 00 a 15). Para migrar de scripts antigos com GS16-GS31, consolidar logicas para o range 00-15.

¶ 🕑 UT analisados a cada 100ms (FW 160B25)

A partir do firmware 160B25 os UTs (User Triggers) sao processados a cada 100ms em vez de 1s. Aumenta a responsividade de detec de eventos rapidos (ex.: pulsos, picos de RPM, transients de tensao).

Para eventos que precisam de mais resposta ainda, usar indices ED212-ED255 (eventos rapidos a 100ms desde FW 154B03).

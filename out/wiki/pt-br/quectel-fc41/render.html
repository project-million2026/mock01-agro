¶ AT+QBLESCAN - Start/Stop BLE Scan (de dispositivos centrais visíveis)

Comando:

AT+QBLESCAN=<scan>[,<timeout>]

AT+QBLESCAN=1

Onde:

0 = Stop

1 = Inicia o Scan BLE, mas não emite dados de Advertising ou dados de resposta da varredura.

2 = Inicia o Scan BLE e gera dados de Advertising, digitaliza os dados de resposta. Não é necessário interromper manualmente a varredura BLE neste modo. 

timeout = Duração da varredura/Scan. Faixa: 500–10000 ms. Deve ser definido quando <scan> for 2.

 

¶ Resposta se scan = 0:

OK 

ou 

ERROR

 

¶ Resposta se scan = 1:

OK
+QBLESCAN:<name>,<address_type>,<address>

ex.:

+QBLESCAN:"VIRTEC_VL8_FC02",0,f7dbbf2747e8

 

¶ Resposta se scan = 2:

OK
+QBLESCAN:<name>,<address_type>,<address>,<adv_data>

ex.:

+QBLESCAN:"VIRTEC_VL8_3333",0,e84727bfdba6,0201061209225649525445435F564C385F3333333322

Onde:

"VIRTEC_VL8_3333" = nome do BLE central visto;

0 = tipo de endereço (0 = endereço público | 1 = endereço randômico);

e84727bfdba6 = Endereço do dispositivo BLE. Comprimento: 12 bytes.

0201061209225649525445435F564C385F3333333322 = Dados de Advertising do BLE visto e dados de resposta.

¶ Advertising Data

Os dados de advertising são decodificados da seguinte forma (da esquerda para a direita):

  • Byte 1 = Comprimento N dos dados a partir do próximo byte
  • Byte 2 = Tipo de Avertisement Data (conforme a tabela de Tipos de dados Comuns na página 13)
  • Próximos N-1 bytes: Conteúdo, de acordo com o comprimento especificado no byte 1

 

Exemplo:

+QBLESCAN:BM31xx-011DAC,0,cc934a011dac,0201060E09424D333178782D303131444143

 

02 = Comprimento (2 bytes)

01 = Tipo de AD (Flags)

06 = Conteúdo (máscara de flags binárias 0b00000110)

 

0E = Comprimento (14 bytes)

09 = Tipo de AD (Complete Local Name)

424D333178782D303131444143 = Conteúdo (convertendo para ASCII: BM31xx-011DAC)

Exemplo:

0201060303E1FF1316E1FFA103640000FFFE00FD1911A03D23AC

Onde:

HEAD

  • 02 = Data Length
  • 01 = Tipo de dados (01 = Flags)
  • 06 = Flag data
  • 03 = Data Length
  • 03 = Tipo de dados (03 = Complete List of 16-bit Service Class UUIDs)
  • E1FF = Dados
  • 13 = Data Length
  • 16 = Tipo de dados (16 = Service Data)
  • E1FF = Head das informações

DATA

  • A1 = Tipo de frame
  • 03 = Versão
  • 64 = Nível de bateria (64 = 100%)
  • 0000 = Eixo X (+0.00g) - Notação 8.8 fixed point
  • FFFE = Eixo Y (-0.01g) - Notação 8.8 fixed point
  • 00FD = Eixo Z (+0.98g) - Notação 8.8 fixed point
  • 1911A03D23AC = MAC Address

 

Exemplos de conversão 8.8 Fixed Point

 

0x0280

Em binário: 00000010.10000000 (positivo)

Parte inteira: 0x02 = 2

Parte decimal: 0x80 = 128 → 128/256 = 0,5

Número convertido: 2,5

 

0xFD80

Em binário: 11111101.10000000 (negativo)

Complemento de 2: 00000010.10000000

Parte inteira: 00000010 = 2

Parte decimal: 10000000 = 128 → 128/256 = 0,5

Número convertido: -2,5

 

0xFFFE

Em binário: 1111111111111110 (negativo)

Complemento de 2: 00000000.00000010

Parte inteira: 00000000 = 0

Parte decimal: 00000010 = 2 → 2/256 = 0,007

Número convertido: -0.007

 

0x00FD

Em binário: 00000000.11111101 (positivo)

Parte inteira: 00000000 = 0

Parte decimal: 11111101 = 253 → 253/256 = 0,988

Número convertido: 0,988

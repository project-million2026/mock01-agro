¶ Níveis homologados na FLESPI

 

¶ Nível I — Monitoramento e Vídeo Telemetria Básica 

 

  • Gerenciar o canal de comunicação com o MDVR, indicando o estado de conectividade (online/offline) e o status de ignição (ligada/desligada). 

Integrado: Para capturar o estado de conectividade, podemos utilizar a requisição:

GET - /gw/devices/{seletor-dev}.

Para capturar o status da ignição, podemos utilizar a requisição:  

GET - /gw/devices/{dev-selector}/telemetry/{engine.ignition.status} com o path “engine.ignition.status”, desta maneira: 

/gw/devices/{dev-selector}/telemetry/engine.ignition.status.

 

 

  • Receber e interpretar pacotes de posicionamento para relatório de localização do MDVR (latitude, longitude, horário e velocidade).

Integrado:Para capturar os pacotes de posicionamento, podemos utilizar a requisição:

GET - /gw/devices/{dev-selector}/telemetry/{telemetry-selector}, com os seguintes paths:

“position.altitude”, ”position.direction”, “position.speed”, “position.timestamp”. Desta maneira: 

/gw/devices/{dev-selector}/telemetry/position.altitude,position.direction,position.speed,position.timestamp.

 

  • Disponibilizar visualização de vídeo e áudio ao vivo de qualquer câmera conectada ao MDVR.

Integrado

 

¶ Nível II — Recuperação de Mídia e Configuração Remota  

 

  • Solicitar e reproduzir vídeos históricos armazenados no cartão SD do MDVR (playback). 

Integrado: Para capturar as mensagens armazenadas pelo dispositivo, podemos utilizar a requisição:

GET - /gw/devices/{dev-selector}/messages

 

  • Executar configurações remotas do equipamento. 

Integrado: Para executar configurações remotas e enviar comandos diversos ao equipamento, podemos utilizar a requisição: 

POST - /gw/devices/{dev-selector}/commands

¶ AG600 + FLESPI

 

¶ ? Acesso inicial ao painel flespi

Para iniciar a integração, primeiro é necessário acessar o painel da plataforma flespi (flespi panel) e realizar o login com seu usuário e senha.

 

Após o login, você verá um dashboard semelhante a este, com os menus necessários para configuração de canais e dispositivos. 

 

A partir desse ponto, já é possível começar as configurações de conectividade do dispositivo Streamax.

 

¶ 1️⃣ Criar um Canal no flespi

O canal é o recurso responsável por receber os dados enviados pelo dispositivo.

 

¶ ? Passo a passo para criar o canal

  1. Acesse a Central de Telemática → Canais → "+"
  2. No campo Name, defina um nome para o canal, por exemplo: Canal Streamax
  3. Em protocol_id, selecione: streamax
  4. Clique em Save para salvar o canal.

Após salvar:

Na aba channels, o canal recém-criado será exibido na lista.

Anote os valores de: HOST e PORT

 

Esses dados serão utilizados na configuração do dispositivo Streamax (endereço e porta do servidor).

 

¶ 2️⃣ Configurar o dispositivo Streamax

Com o canal criado, agora é necessário configurar o dispositivo Streamax para enviar dados ao flespi.

  1. Abra o aplicativo móvel Veyes (disponível para iOS e Android).
  2. Siga o passo a passo de configuração detalhado napágina correspondente ao modelo do equipamento.

 

¶ 3️⃣ Criar um dispositivo no flespi

Depois de configurar o canal e o equipamento, é preciso criar o dispositivo lógico no flespi para visualizar e organizar os dados.

 

  • Acesse: Telematics hub → Devices → "+"
  • Defina um nome para o dispositivo (por exemplo: Streamax Ônibus 01).
  • Em device_type_id, selecione o modelo adequado, por exemplo: AD Plus 2.0 (ou outro modelo Streamax apropriado).
  • Em Configuration → Identification preencha o número de série (serial) do dispositivo. (você pode encontrá-lo na aba "Registros e Mensagens" do canal após a conexão ou no aplicativo Veyes)
  • Clique em Save para salvar o dispositivo.

 

¶ 4️⃣ Verificar se a integração está funcionando

Com canal e dispositivo configurados, o próximo passo é confirmar se os dados estão chegando corretamente.

  1. No canal:

    -  Acesse a aba “Logs & Messages” do canal
    - Verifique se existem registros de conexão do dispositivo
     
  2. No dispositivo:

    -  Acesse a aba “Logs & Messages” do dispositivo
    -  Confira se as mensagens recebidas estão sendo exibidas
     
  3. Status de conexão:

    - Verifique se o dispositivo aparece como conectado
    - Normalmente isso é indicado por um ícone verde ao lado do dispositivo

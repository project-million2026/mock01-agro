¶ Configurações iniciais

 

¶ Definição do modelo de equipamento

São utilizados os comandos VS abaixo para definir o tipo de equipamento ou placa testado no PEM HOST.

Comandos:

>VS08,A1,8< // Board VLx
>VS16,54,8< // DeviceType VLx

Onde:

  • 8 = Hardware VL8 (6 = VL6 | 8 = VL8)

 

¶ Detecção de placa presente

Comando para configurar a identificação de placa.

>VS08,A0,0< // BOARD DETECT OFF

Onde:

  • 0 = detecção de placa presente desativada (para teste com cabo) (0 = detecção de placa presente desativada | 1 = detecção de placa presente ativada)

 

¶ Baudrate de comunicação com porta serial VL

Comando para configurar o Baudrate de comunicação com porta serial VL.
Comando:

>VS32,76,115200< // BAUDRATE

Onde:

  • 115200 = baudrate.

¶ Sleep VL no final de teste 

Existe a opção do PEM enviar comando de Sleep forçado no final do teste, este recurso é muito utilizado quando o equipamento é testado já com a bateria interna conectada.

>VS08,9A,1<

  • 1 = ativa o envio do comando >VSLP_SLEEP0000< ao final do teste, se 0 é desativado este envio.

¶ PEM0020 - Teste de entradas digitais

Os testes das entradas digitais são configurados pela máscara de bits do comando abaixo.

Comando:

>VS08,7C,23< 

Onde: 

  • 23 = máscara para ativar teste de IOs
    • 23 decimal = 00100011 binário
      • 1 = 0x01 IN0 TEST
      • 1 = 0x02 IN1 TEST
      • 0 = 0x04 IN2 TEST
      • 0 = 0x08 IN3 NO TEST (RX)
      • 1 = 0x10 IN4 TEST

 

¶ Teste de IOs para VL8

>VS08,7C,23< // IOtest VL8 (17 HEXA = 23 EM DECIMAL)

  • 01 0x01 IN0 TEST
  • 02 0x02 IN1 TEST
  • 04 0x04 IN2 TEST
  • 08 0x08 IN3 NO TEST (RX)
  • 16 0x10 IN4 TEST

 

Obs.: o PEM envia um VS19_ENA0 para desabilitar a CAN antes do teste.

Obs.: no VL8, deve-se configurar também o valor limite para se detectar 0 e 1 em IN0 através do comando abaixo.

>SIS10SIT00003003_V1:2761< //TENSÃO PARA DETECTAR 0/1 NA IN00 - AGREGADO NO FW 3.26

Onde:

  • 2761 = TENSÃO PARA DETECTAR 0/1 EM IN00 (em Volts x 1000: 2761 =  2,761V)
     

 Exemplo de resultado do teste:

>PEM0022_INS=1_PASS;ID=PH50;#0011;*1A<
>PEM0025_IN0=0_PASS;ID=PH50;#0012;*7C<
>PEM0028_IN1_NOT_TESTED;ID=PH50;#0013;*70<
>PEM0031_IN2_NOT_TESTED;ID=PH50;#0014;*7C<
>PEM0034_IN3_NOT_TESTED;ID=PH50;#0015;*79<
>PEM0037_IN4_NOT_TESTED;ID=PH50;#0016;*7E<

 

¶ Teste de IOs para VL6

>VS08,7C,01< // IOtest VL6 (01 HEXA = 01 EM DECIMAL)

  • 01 0x01 IN0 TEST
  • 02 0x02 IN1 TEST
  • 04 0x04 IN2 TEST
  • 08 0x08 IN3 NO TEST (RX)
  • 16 0x10 IN4 TEST

 

VL6_CAN / VL6_CAN_BT

>VS08,7C,01< // IOtest

  • 01 0x01 IN0 TEST
  • 02 0x02 IN1 NO TEST (CAN)
  • 04 0x04 IN2 NO TEST (CAN)
  • 08 0x08 IN3 NO TEST (RX)
  • 16 0x10 IN4 NO TEST (VL6)
     

>VS08,8A,1< // CANtest ON

 

VL6_NO-CAN (BR VL6 ECO) / VL6_NO-CAN_NO-BT (BR VL6 FULL)

>VS08,7C,07< // IOtest

  • 01 0x01 IN0 TEST
  • 02 0x02 IN1 TEST
  • 04 0x04 IN2 TEST
  • 08 0x08 IN3 NO TEST (RX)
  • 16 0x10 IN4 NO TEST (VL6)

 

>VS08,8A,0< // CANtest OFF

¶ Carregar script de testes

  1. Apagar as configurações enviando >CLN12345678<
  2. Carregar o script correspondente ao equipamento que será testado
  3. Reiniciar o PEM (basta desconectar e reconectar o cabo USB)

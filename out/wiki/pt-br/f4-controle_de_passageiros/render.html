Os equipamentos que têm o tipo de máquina definido como 0 (TCFG3,0) se comportam como entrada de controle, como um totem para controlar a entrada de veículos.

  • Eles sempre enviam broadcast de conectividade e sincronismo (eles não enviam broadcast de permissões - tipo 7).
  • Ao ler um cartão, ele primeiro analisa o código em relação à tabela do veículos.
  • Se ele não conseguir encontrar o código na lista de veículos, dispara o CR09++ e verifica o banco de dados para varrer as autorizações de entrada pessoal.
  • Se mesmo assim não puder encontrá-lo, ele aciona o disparador CR10++.
  • Se o cartão for encontrado e houver apenas pessoal embarcado no veículo em Verde, dispara CR11++.
  • Se houver alguém embarcado em Vermelho, dispara CR12++.

 

¶ Consultas

 

¶ Lista de Permissões 

O comando GRPxx pode solicitar a lista de permissões por índice xx (00..63)

A mesma consulta pode ser realizada por ID do equipamento na rede, acrescentando o ID do equipamento GRP_idid para solicitar a lista de permissões por índice xx (00..63) ou por ID de equipamento (idid).

Acrescentando o caractere P, pode se pedir os presentes

Exemplo:

 >GRPPxx<

 >GRPP_1234<

Onde:

xx = índice da lista de permissões [00 a 63].

1234 = ID do equipamento.

 

 

 

- Adicionado> GRPxx_SET, idid, Green, Red, driver <para inserir uma permissão na lista.

Se xx não for indicado, um novo é adicionado ou o registro com o nível de sinal mais baixo é acionado.

 

- Adicionado GRPxx_CLR e GRP_idid_CLR para deletar um registro na lista.

 

¶ Registros em Memória

 

Existe a possibilidade de escrever vários registros na memória com o comando VSRT.

>VSRT0,"aaaaa""bbbbb""ccccc"<

Este comando escreve:

  • string aaaaa no registro 0 (VSRT00)
  • em 1 (VSRT01) bbbbb 
  • em 2 (VSRT02) ccccc.

Eles também podem ser escritos como texto longo, quando os 16 caracteres de um registro forem preenchidos, ele continua no próximo registro.

> VSRT0,0123456789abcdefghijklmnopqrstuvwxyz<

 

¶ Lista de Cartões

Quando criada uma região RM (definida com o comando VSRMxx) para lista de cartões embarcada, por exemplo região RM10, o formato é fixo e reserva 10 colunas de 16 caracteres para cada cartão.

Exemplo:

>VSRM10,0,300000,499000< // região RM10 para lista de cartões

Onde:

10 = região RM10;

300000 = Ponto inicial da memória;

499000 = Ponto final da memória.

 

Assim um exemplo de registro para esta memória:

>VSRT300000,"1""12345678""3,1""35988635""JOAO""SOARES"ABC"DDD"FF"AA"<

1= dado armazenado no registro 300000

12345678= dado armazenado no registro 300001

3,1= dado armazenado no registro 300002

35988635= dado armazenado no registro 300003

JOAO= dado armazenado no registro 300004

SOARES= dado armazenado no registro 300005

ABC= dado armazenado no registro 300006

DDD= dado armazenado no registro 300007

FF= dado armazenado no registro 300008

AA= dado armazenado no registro 300009

Assim cada cadastro na tabela ocupa 10 registros na memória (VSRT300000 até VSRT300009). 

 

¶ Região de memória definida para cartões

Para indicar a região de memória onde foram cadastrados os cartões, deve-se informar pelo comando TCFG00:

>TCFG00,10<

Onde:

10 = Região RM definida para lista de cartões.

 

¶ Lista de estados dos Cartões

Pode ser definida uma região RM para criar uma tabela indicando o estado dos cartões cadastrados.

Exemplo:

>VSRM11,0,70000,70999< // região RM11 para lista de estados dos cartões

Onde:

11 = região RM10;

70000 = Ponto inicial da memória;

70999 = Ponto final da memória.

 

¶ Região de memória definida estado dos cartões

Para indicar a região de memória onde foram cadastrados os estados dos cartões, deve-se informar pelo comando TCFG01:

>TCFG01,11<

Onde:

11 = Região RM definida para lista dos estados de cartões.

 

¶ Manipulação de Cartões

 

¶ PCR_DRV

Define a próxima leitura do cartão como Motorista. A próxima leitura do cartão será armazenada no vetor 0 (motorista).

Se o contador do cartão estiver em zero, é incrementado em 1.

Após a leitura da placa DRV, passa para o valor 1, indicando que o driver já está ativo.

 

¶ PCR_LIST:a,b

Pede uma lista de cartões embarcados no intervalo a, b. O intervalo máximo é 50.

Retorna a lista de códigos de cartão e seus status correspondentes.

O Motorista é adicionado 100 ao estado.

Cartões com status 0 não são listados.

Os cartões não encontrados no RM10 são exibidos com o status 99.

A lista máxima retornada é de 15 cartões para não exceder o comprimento da string.

 

¶ PCR_CLR_LIST:a,b

Exclui a lista de cartões embarcados no intervalo a, b.

 

¶ PCR_TIMEOUT_LIST:a,b

Reinicializa os contadores de entrada / saída no intervalo a, b.

 

¶ PCR_REGxx

Imprime os dados do registro X da lista de remessa. X pode ser uma variável com valor 0..50.

 

¶ PCR_REG_LAST

Imprime os dados do último cartão lido.

 

¶ PCR_REGxxxx_ST

Imprime a descrição do estado indexando a tabela RM11 com o estado numérico (coluna 1 do cartão).

 

¶ Variáveis ​​para impressão da lista de cartões na tela VIRCOM

>VCRxxyy<

Onde:

xx = Índice de registro [00..63, 99 = último cartão lido]

yy = Coluna correspondente ao cartão [0..15, 99 = imprime texto do estado do cartão indexando em RM11.

 

¶ Buscar texto na lista

>VGCS,a,b,j,texto<

Encontre o texto na lista a..b saltando de j em j registros.

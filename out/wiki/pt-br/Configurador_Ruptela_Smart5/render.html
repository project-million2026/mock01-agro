¶ Uma Breve informação sobre o configurador

O configurador avançado é usado para configurar os dispositivos de rastreamentos da Ruptela e todas as suas funcionalidades, permitindo que você a melhor solução para suas necessidades.

 É permite que você faça os seguintes casos

  • Crie um novo arquivo de configuração
  • Editar um arquivo de configuração existente
  • Envie um arquivo de configuração para o dispositivo
  • Carregue um arquivo de configuração existente do seu dispositivo
  • Modificar todas as funcionalidades do dispositivo
  • Atualize o firmware do dispositivo

 

¶ Tela Inicial 

 

1 = Barra de Menu 

2 = Informações do arquivo de configuração

3 = Comunicação com o equipamento

4 = Informações do dispositivo conectado

5 =  configuração dos parâmetros do script

 

¶  1-Barra de Menu 

¶ Barra de Arquivos

New CFG = inicia a criação de uma nova configuração.

Open default CFG = Carrega uma configuração padrão com valores predefinidos.

Save CFG = Salva a configuração em sua maquina

Open CFG = Carrega um arquivo de configuração predefinido. 

Save DIFF CFG =  Cria um arquivo DIFF que é usado para alterar apenas um ou alguns parâmetros específicos sem a necessidade de enviar a configuração completa para o dispositivo. É útil ao atualizar dispositivos remotamente

Open DIFF CFG = Abre um arquivo de configuração DIFF salvo

Generate SMS = Gera um comando SMS para configurar o dispositivo FM remotamente.

Exit = Fechar o configurador

Salvar DIFF CFG salvará apenas as alterações feitas na configuração inicial (uma configuração predefinida carregada).

Ao clicar em Exit, o configurador será fechado imediatamente, sem solicitação de confirmação. Alterações não salvas serão perdidas.

 

¶ Barra de Ferramentas

Language = Seleciona o idioma disponível (inglês ou russo).

Live IO data = Abre uma janela exibindo todas as entradas e saídas habilitadas, juntamente com seus respectivos valores.

Set Virtual odometer = Abre uma janela onde você pode definir um novo valor para o odômetro virtual (em metros)

Sensor marker = Abre uma janela onde você pode marcar sua temperatura e umidade

 

¶ 2- Informação do arquivo de configuração 

Configuration source = De onde a configuração foi carregada

FM device FW version =Versão atual do firmware do dispositivo. Exibida apenas quando a configuração é carregada diretamente do equipamento.

Advanced Configurator version = A versão do configurador na qual a configuração atualmente carregada foi feita.

Target device = O modelo do dispositivo para o qual esta configuração foi criada ou a partir do qual foi carregada

CFG Tag = Campo de entrada onde podem ser salvas informações adicionais sobre o arquivo de configuração. O comprimento máximo é de 32 caracteres. Somente caracteres ASCII são permitidos. 

Last edited = A data e hora exatas em que a configuração foi editada pela última vez

¶ 3-Selecionar o equipamento

 

Na parte esquerda do configurador, tem as informações do hardware(IMEI, Firmware, Hardware e versão), embarcar o arquivo (Send CFG), embargar versão de firmware (Send FW), ver a ultima configuração (Get CFG), tem como selecionar o dispositivo e a versão dele.

No campo de selecionar equipamento, o Smart5 que fornecemos é o Smart5-CAN-LTE-LA

Recomendamos ativar o campo SIM status, pois ele representa que o NanoChip está conectado ao equipamento

No primeiro campo selecione a porta COM que está o dispositivo e clique em connect

Send CFG = embarcar o script configurado

Get CFG = ver o ultimo script embarcado

Send FW = enviar atualização de firmware.

Nos campos de baixo(entre Send FW e as informações do hardware) selecionamos o equipamento e a versão dele, Smart5 e Smart5-CAN-LTE-LA)

 

¶ 4 - Informações sobre o hardware

 

IMEI = O código IMEI exclusivo do dispositivo.

FW = versão do firmware

BL = A versão do bootloader

HW = versão do hardware 

Status do SIM = Exibe o status atual do cartão SIM conectado e permite o bloqueio

O status correto do SIM só será exibido se houver uma fonte de alimentação externa conectada.

Se a caixa de seleção de status do SIM não estiver ativada, a janela "Inserir PIN" não será exibida.

Se o código PUK incorreto for inserido dez vezes seguidas, o dispositivo será bloqueado permanentemente, exigindo um novo cartão SIM.

Se o PIN incorreto for inserido três vezes seguidas, o cartão SIM será bloqueado, exigindo o código PUK, fornecido pela operadora do serviço

¶ 5- Configuração dos parâmetros para o script

Configurações principais do Smart5

 

¶ Password

É obrigatório a configuração de uma senha no script nos campos de Configuration Password e Authorized numbers. A quantidade de caracteres é de 1 a 16  

Em Authorized numbers apenas preencha os dois primeiros campos.

 

¶ APN

O Smart5 suporta apenas um Nano Chip, portanto é possível configurar somente uma APN.

No campo de APN settings apenas você preenchendo os campos;

Name =  APN do Chip

User = usuário da APN

Psw = senha da APN

Também é possível utilizar a opção AutoAPN.

 

¶ Configurações do servidor de destino

É possível configurar dois servidores, utilizando protocolo UDP ou TCP.

IP1 = numero do servidor 1(aaa.bbb.ccc.ddd) em numeros

Port 1 = a porta do servidor 1

IP2 = numero do servidor 2 (aaa.bbb.ccc.ddd) em números

Port 2 = a porta do servidor 2

A opção de Two servers é apenas para transmitir repetidamente os mesmos dados para ambos os servidores 

Em Periodical redirect é utilizado para redirecionar periodicamente o dispositivo para um servidor específico, a fim de iniciar atualizações automáticas quando um software de rastreamento de terceiros for utilizado.

DMP Settings = Esta funcionalidade permite definir como o dispositivo se conectará à plataforma de gerenciamento de dispositivos (DMP) para receber atualizações remotas de firmware e/ou configuração. A conexão com a plataforma de gerenciamento de dispositivos está habilitada por padrão.

Por padrão, está configurado para se conectar à plataforma duas vezes por dia, é possível customizar a quantidade de conexões diárias pela plataforma DMP com o Smart5.

¶ 2 - Data sending

Essas configurações permitem definir a frequência e as condições dos dados enviados ao servidor. A caixa de seleção Enable deve estar marcada para habilitar os parâmetros de envio de dados. 

Min Records = O número mínimo de registros necessários para estabelecer uma conexão com o servidor.

Period = Com que frequência o dispositivo verificará o parâmetro "Min Records" 

Link Timeout = Por quanto tempo o dispositivo manterá a conexão antes de fechá-la, após a conexão ter sido estabelecida com sucesso.

¶ 3 -Data collection

Essas configurações permitem definir o tempo(em segundos) e as condições dos dados coletados.

Time without engine = Determina com que frequência os registros são feitos quando o motor está desligado

Send only record header if triggered by time, distance or course  = Se ativado, envia apenas o cabeçalho do registro para reduzir o consumo de dados. O cabeçalho do registro contém apenas dados básicos de tempo, posição e velocidade.

GPS navigation filtering = Utilizada para eliminar coordenadas GPS imprecisas. Esta funcionalidade pode ser Ativado e modificado clicando no botão "Customize".

Coefficients = Utilizado para coletar registros, além de outros parâmetros, para uma análise mais precisa da rota do veículo.

 Engine = Com qual fio irá detectar se o motor está ligado(fio de ignição).

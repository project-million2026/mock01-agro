¶ Configurações Gerais

Principais configurações para o VIRLOC 6

 

¶ ? Controle de Energia

Desligando dispositivos

>VSLP0000,xxxx<

Ativando dispositivos

>VSLPFFFF,xxxx<

Onde:

xxxx = Bitmap dos dispositivos para ligar/desligar.

Bitmap dos dispositivos:

  • 0001 = GNSS;
  • 0002 = MDM;
  • 0004 = BT;
  • 0008 = Alimentação MDM;
  • 0020 = Fonte de alimentação;
  • 0040 = Pullup entrada 0;
  • 0080 = Pullup entradas IN01, IN02, IN03 e saída OUT1;
  • 0100 = Eventos;
  • 0800 = Carregador de bateria.

 

 

¶ ? Calibração das Entradas Analógicas

Para calibrar uma entrada analógica, conecta-se uma fonte de tensão ao VIRLOC através da entrada desejada e utiliza-se o seguinte comando:

>SOMpppppppp,X,1000,1000<

Onde:

pppp = Senha geral do equipamento

X = Número da Entrada analógica (0 - Vmain, 1 - Bateria interna, 2 - Entrada analógica IN0)

1000 = Fator de calibração

Recomenda-se a utilização de um equipamento de medição de tensão como multímetro ou voltímetro uma vez que a tensão da fonte conectada na entrada analógica é desconhecida.

 

Consulta da tensão na entrada analógica:

>QOMxx<

Onde:

xx = Número da Entrada analógica

Retorno: 

>ROM0,1000,1000 1359;ID=6091;#819F;*20<

Onde:

1359 = Tensão lida

 

¶ ? Configurações do Modem

O VIRLOC 6 possui dois slots para SIM Card, o slot superior é o primário e o inferior é o secundário. Isso significa que o VL6 realizará o escaneamento primeiramente pelo slot superior e depois pelo inferior, assim, é recomendado utilizar o slot superior se for utilizar apenas um SIM Card para obter maior rapidez em casos de reconexão.

As configurações dos SIM Cards podem ser realizadas conforme descrito abaixo:

 

¶ APN primária (Slot superior)

>SIS04AT+CGDCONT=1,"IP","generica.claro.com.br"<

>SIS16"claro""claro"<

Onde:

generica.claro.com.br = APN do SIM1

claro = Usuário do SIM1

claro = Senha do SIM1

 

¶ APN secundária (Slot inferior)

>SIS14AT+CGDCONT=2,"IP","zap.vivo.com.br"<

>SIS17"vivo""vivo"<

Onde:

zap.vivo.com.br = APN do SIM2

vivo= Usuário do SIM2

vivo= Senha do SIM2

 

¶ Consulta de Strings do Modem

>QISxx<

Onde:

xx = número da String

 

? Por exemplo, para consultar APN do SIM1:

>QIS04<

Resposta:

>RIS04AT+CGDCONT=1,"IP","allcom.vivo.com.br";ID=0486;#800C;*0F<

 

¶ ? Configuração do IP de destino

O VIRLOC 6 tem a possiblidade de configuração de dois IPs de destino para as mensagens, Socket 0 e Socket 1 (IP0 e IP1).

Para consultar os destinos, enviar o comando:

>QIPx<

Onde:

x = 0 → IP 0 (Socket 0)

x = 1 → IP 1 (Socket 1)

 

Configuração:

>SIPx,aaa.bbb.ccc.ddd,ppppp,m,F<

Onde:

x = Número do endereço IP [0 ou 1]

aaa...ddd= Endereço IP destino

ppppp= Porta de destino

m = Tipo de Socket - 0 = UDP

F = Socket (IP) alternativo (1 = Socket 1, F = desabilitado)

 

¶ ? Configuração de destino DNS

Para configurar o VIRLOC 6 para envio em um DNS, o IP 0 e o IP1 devem ser preenchidos com 0.0.0.0, ou outro IP determinado opcionalmente, e a porta utilizada nos respectivos DNS conforme o exemplo abaixo:

>SIP0,0.0.0.0,ppppp,0,1<

>SIP1,0.0.0.0,ppppp,0,F<

Onde:

ppppp = porta de destino

 

Os DNS podem ser consultados com os comandos abaixo:

>QIS24< → Consulta DNS de IP0

>QIS25< → Consulta DNS de IP1

 

Configuração:

>SIS24www.virtec.com.ar< → Configura DNS IP0

>SIS25< → Desabilita DNS IP1

 

⚠️ Ao alterar alguma configuração de servidor o ideal é realizar o reinicio do modem com o comando SVP0 2.

 

¶ ? Reset do modem

Para reiniciar o modem GPRS, utiliza-se o comando:

>SVP0 2<

 

¶ ? Configurações de Ignição

O VIRLOC 6 permite três tipos de configuração da ignição: Física, Virtual e por CT.

 

¶ Ignição Física

É possível utilizar a IN0 como entrada de ignição física. Para isso deve-se configurar os seguintes parâmetros:

>VS16,00,1100<

>VS16,02,9999<

Onde:

1100 = tensão na IN0 para detectar ignição ligada (11,00V)

9999 = desativa ignição virtual

 

¶ Ignição Virtual

É possível utilizar a tensão de alimentação principal para detectar a ignição. Para isso, deve-se configurar os seguintes parâmetros:

>VS16,00,9999<

>VS16,02,1350<

Onde:

9999 = desativa ignição física

1350 = tensão da alimentação para detectar ignição ligada (13,50V)

 

¶ Ignição por CT

É possível utilizar um CT que contenha o valor de RPM (por exemplo) para detectar ignição. Para isso, deve-se configurar os seguintes parâmetros:

Ignição por CT:

>VS16,00,300<

>VS16,02,1<

Onde:

300 = valor mínimo para definir Ignição ligada

1 = CT para utilizar como referencia para ignição

 

¶ Tempo para manter ignição ativa

>VS16,2C,200<

Onde:

200 = reconhece ignição desligada 20 segundos após ter sido desativada

 

¶ IN00 para digital acionada com GND

>VS16,120,205<

Onde:

205 = Tensão para reconhecer acionamento de GND

 

¶ ? Tensão da Alimentação Principal

>VS16,04,1080<

Onde:

1080 = Tensão mínima para reconhecer alimentação principal conectada (10,80V)

 

¶ ⚡️ Fonte de Velocidade

O VIRLOC 6 pode utilizar o GPS ou um CT como fonte do dado de velocidade. Para definir essa configuração, deve-se utilizar o seguinte parâmetro:

>VS08,07,255< OU >VS08,07,xx<

Onde:

255 = pega velocidade do GPS

xx = pega velocidade do CTxx

 

⚠️ O valor da velocidade no CT deve ser multiplicado por 1000 para ser inserido corretamente.

 

¶ ⚙️ Comando SSO

Os dados são armazenados em uma memória RAM (volátil). O equipamento trabalha com as informações na RAM e periodicamente faz um backup na memória FLASH (não volátil) automaticamente ou quando executando este comando.

O firmware dos VIRLOCS realiza salvamentos automáticos em flash de acordo com as ocorrências descritas abaixo:

  • A cada 10Km percorridos;
  • Ao perder a alimentação principal (quando a bateria de backup disponível para manter o equipamento ligado);
  • Ao desligar ignição;
  • Ao receber um comando VSLP;
  • Ao detectar velocidade abaixo de 5 km/h se estava com velocidade superior por mais de 3 minutos;
  • Ao entrar no ciclo de sleep (modo KO).

Quando o equipamento ficar sem energia por um longo período, ele irá recuperar o estado em que se encontrava, quando o equipamento for reiniciado, caso tenham sido salvos, são recuperados automaticamente.

O comando também pode ser executado manualmente ou por meio de uma ação de evento, porém recomenda-se cautela, visto que escrever dados em flash frequentemente pode acarretar dano à memória. Além disso, a execução deste comando é limitada pelo Virloc, podendo ser feita até 1 milhão de vezes.

 

>SSO MSxxx XSyyy<

Este comando salva:

  • Os valores dos contadores estatísticos.
  • Os valores dos contadores de tempo CC.
  • Os valores dos contadores numéricos CT (somente CT00 a CT99).
  • Os ponteiros do LOG de dados (de LOG e LG2).
  • Estado das saídas.
  • Estado das flags (SG).
  • Valor atual do número de mensagens transmitidas.
  • Valores do contador de tráfego IP.

 

¶ ? Senha Geral

A senha geral padrão do VIRLOC 6 é 00000000, podendo ser alterada através do procedimento abaixo.

Defina uma senha geral de 8 dígitos. Deve ser enviada 3 vezes com um indicador de sequência.

>SPEs,aaaaaaaa,bbbbbbbb<

Onde:

s = Numero de sequência (1, 2, 3)

aaaaaaaa = Senha atual

bbbbbbbb = Nova senha

 

¶ ? Trocando ID

O ID é uma cadeia de 4 caracteres alfanuméricos que representa o equipamento e permitirá reconhecê-lo dentro da frota. De fábrica, é configurado um ID correspondente aos quatro últimos caracteres do serial number do equipamento, mas pode ser modificada pelo usuário.

>VSIDpppppppp,DDDD<

Onde:

pppppppp = Senha geral

DDDD = Novo ID

 

⚠️ Alterar o ID elimina toda a memória do buffer FIFO/LIFO.

 

¶ ? Reset para a configuração de fábrica

Apagando as configurações:

>CLNpppppppp<

 

Apagando os eventos:

>CLNpppppppp_EVT<

 

Apagando as configurações CAN:

>CLNpppppppp_CAN<

 

Apagando o buffer FIFO/LIFO e LOG:

>GFFEpppppppp<

 

Limpeza completa do equipamento:

>CLNpppppppp_FULL<

 

⚠️ Nenhum dos CLNs funcionam via servidor.

pppppppp = password geral (padrão 00000000)

¶ Introdução

O Vircom 07 é um equipamento de telemetria completo que conta com tela e teclado integrados e permite a criação de telas customizáveis para aplicações diversas. Esta documentação também se aplica ao Vircom 05 (descontinuado), porém pode haver incompatibilidade com recursos de firmware mais recentes. Em caso de dúvidas quanto a compatibilidade de algum recurso, consulte a nossa equipe de suporte.

 

O display conta com uma tela LCD de 128x64 pixels que se divide em:

  • 8 linhas [0 a 7]
  • 121 colunas [0 a 120]

 

¶ Telas

Em uma tela não programada, o Vircom irá exibir a mensagem “BAD SCREEN CONFIGURATION”, indicando que não há configurações para a tela atual. A tela exibida inicialmente sempre será a tela 00.

 

O firmware disponibiliza 61 telas programáveis, com índices de 00 a 60. Para acessar uma tela, utiliza-se o comando abaixo.

>TSSCxx<

Onde xx é o índice da tela. Esse comando pode ser enviado diretamente para o Vircom ou também pode ser utilizado na ação de um evento, por exemplo:

>SED001 IN07++ +- SGN NN {TSSC03}<

 

Para criar uma tela, basta iniciá-la com o comando TSSB, e no mesmo comando podem ser inseridos os elementos gráficos que serão desenhados na tela assim que ela for chamada e executadas ações de configuração.

 

Criação da tela:

>TSSB00{TSBL90}{TTXT,40,03,NEWTEC}<

 

Consulta:

>TQSB00<

 

Remoção da tela:

>TSSB00U<

Onde:

  • 00 = Índice da tela [00 a 60]
  • {TSBL90}{TTXT,0,03,NEWTEC} = Configurações e elementos gráficos

 

Também é possível consultar a tela atual do VIRCOM 7

 

Consulta:

>QSC<

Resposta:

 

>ACK_RSC10;ID=1234;#A2B9;*2F<

 

Onde:

00 = Índice da Tela atual

¶ Fontes

Existem três formatos de fontes possíveis no Vircom, que são:

  • 0 = Fonte padrão (ocupa 2 linhas)
  • 1 = Fonte destacada (ocupa 2 linhas)
  • 2 = Fonte menor (ocupa 1 linha)
  • 4 = Fonte grande (ocupa 4 linhas)

Para modificar a fonte, utiliza-se o comando TSFN, a partir do qual todo texto escrito terá a fonte especificada.

 

Exemplo:

>TSSB00{TSFN1}{TTXT,40,00,NEWTEC}{TSFN2}{TTXT,34,02,TELEMETRIA}{TSFN4}{TTXT,27,04,000}<

 

Resultado:

 

Também é possível destacar somente um trecho de texto utilizando os caracteres g e f:

  • 0 = Fonte padrão
  • g = Início do destaque
  • f = Fim do destaque

 

Exemplo:

>TSSB00{TSFN0}{TTXT,00,00,NORMAL gDESTACADOf}<

¶ Menus

Os menus são textos contornados por um retângulo, normalmente utilizados para criação de itens num menu de seleção indicando com setas as teclas correspondentes.

Exemplo:

>TSSB00{TSFN0}{TTXM,70,00,ITEM1-m}{TTXM,70,03,ITEM2-m}{TTXM,70,06,ITEM3-m}<

Onde:

  • TTXM = Ação de criação de menu
  • 70 = Posição horizontal (coluna em que o texto será inserido)
  • 00 03 06 = Posição vertical (linha onde o texto será inserido)
  • ITEM1 = Texto associado
  • -m = “Seta” formada por hífen e caractere maior (->)

¶ Entrada de dados

Para criar um campo de entrada de dados, o primeiro passo é inserir no comando de criação da tela a configuração TSDE exemplificada abaixo.

>TSSB04{TTXT,02,01,INSIRA O CoDIGO}{TSDE01,20,03,1000010----------}{TTXM,70,06,ENTER-m}{TXM,00,06,l-LIMPA}<

Onde:

  • 01 = Índice do campo [00 a 15]
  • 20 = Posição horizontal (coluna em que o primeiro espaço será inserido)
  • 03 = Posição vertical (linha onde o primeiro espaço será inserido)
  • 1 = Modo normal
  • 0000 = Posição do buffer para começar a inserir os dados (normalmente 00)
  • 10 = Quantidade de caracteres no campo
  • ---------- = Preenchimento inicial do campo

 

Resultado:

 

Consulta:

>TQDE01<

 

Retorno:

>ACK_RDE01,020,03,10000101234567890<

 

Para apagar o último caractere inserido:

>TSSE0400 CB02++ +- SGN NN {TBDE01}<

 

Onde:

  • 01 = Índice do campo [00 a 15]
  • 020,03,1000010 = Configurações feitas para o campo
  • 1234567890 = Dados inseridos (inicialmente terão o preenchimento definido no comando TSDE)

 

 

¶ Eventos de tela

Os eventos de tela seguem uma sintaxe semelhante aos eventos normais em XVM. Cada tela pode ter até 16 eventos associados, que são processados somente enquanto a tela está sendo executada.

 

Criar um evento de tela:

>TSSE0000 CB13++ US00++ SGN NN {TIC}<

 

Consulta:

>TQSE0000<

 

Remoção:

>TSSE0000U<

 

Onde:

  • 00 = Índice da Tela [00 a 60]
  • 00 = Índice do Evento de tela [00 a 15]
  • CB13++ = Disparador (podem ser usados até 2 disparadores com operadores lógicos && ou ||)
  • US00-- = Condicional (podem ser usadas até 2 condições com operadores lógicos && ou ||, ou nenhuma condição com +-)
  • SGN NN* = Destino e formato de mensagem
  • {TIC} = Ações diversas (opcional)

* Nos eventos de tela os formatos de mensagem disponíveis são apenas AC e AX (vide Reportes AC e AX). Para enviar reportes com formato definido e reportes configuráveis utilizar Flags ou CTs na ação dos eventos de tela.

¶ Disparadores de botões

Cada um dos botões do Vircom possui um disparador específico que pode ser usado nos eventos de tela como disparadores. Existem também disparadores que podem ser usados nos eventos padrão (a partir do firmware 154B20), conforme a tabela a seguir.

 

 

Exemplo:

>TSSB00{TSFN0}{TTXM,70,00,ITEM1-m}{TTXM,70,03,ITEM2-m}{TTXM,70,06,ITEM3-m}<
>TSSE0000 CB15++ +- TRM AC {ITEM1 SELECIONADO<
>TSSE0001 CB14++ +- TRM AC {ITEM2 SELECIONADO<
>TSSE0002 CB13++ +- TRM AC {ITEM3 SELECIONADO<

No exemplo acima, é desenhada uma tela com indicações para os botões de seleção. Cada botão pressionado irá disparar um dos eventos e um reporte será enviado para o terminal.

 

¶ Disparador configurável

Funcionam de forma similar ao UT nos eventos do Virloc, mas devem ser declarados ou consultados sempre como TSUT ou TQUT. São disparadores configuráveis que capturam um determinado valor retornado em uma consulta, e disparam quando este entra ou sai de uma faixa, também podendo ser utilizados como condicionais.

 

Configuração:

>TSUTxx,CMD,from,len,valA,valB<

Onde:

  • xx = número do disparador configurável [00..63].
  • CMD  = Comando a executar;
  • from = Capture a partir da posição de resposta ao comando;
  • len = Capture len caracteres (comprimento);
  • valA = Valor inferior para a comparação;
  • valB = Valor superior para a comparação.

 

Exemplo:

>TSUT00,QTT,33,3,10,20<

>TSSE0000 UT00-- +- SGN NN {TBEP0060050501000005}<

>TSSE0001 CK00++ UT00-- SGN NN {TBEP0020050501000010}<

¶ Contadores de tempo

De forma semelhante aos contadores CC do Virloc, existem 16 contadores dedicados do Vircom. Podem ser configurados no comando de criação da tela ou na ação de um evento.

>TSSB01{TTXT,70,00,TELA 01}{TSCC0130}<

>TSSE0101 CC01++ +- TRM AC {FIM DA CONTAGEM<

Onde:

  • 01 = Configura o CC01 [00 a 15]
  • 30 = Tempo em segundos x10 (30 = 3.0s)
  • CC00++ = Dispara quando a contagem finaliza

 

¶ Acionamento do buzzer

Para controlar o buzzer interno do Vircom, existe o comando TBEP. Podem ser criados diversos padrões de beeps através das configurações abaixo. O comando pode ser usado tanto nas ações dos eventos de tela quanto nos eventos padrão do Virloc.

>TBEP0020050501000050,F1100,F1200,F1300,F1400<

Onde:

  • 002 = Repetições (002 = uma repetição)
  • 005 = Tempo soando em centésimos de segundo (005 = 0.05s)
  • 050 = Largura de pulso
  • 1000 = Frequência do som
  • 050 = Tempo entre repetições em centésimos de segundo (050 = 0.50s) - Parâmetro opcional
  • F1100,F1200,F1300,F1400 = Frequências adicionais: até 4 frequências para repetir os beeps configurados em tons diferentes

 

¶ Acionamento de LEDs

O Vircom 7 possui dois LEDs localizados atrás do botão “Emergência” que podem ser utilizados para sinalização e são acionados conforme indicado abaixo. Podem ser usados nas ações dos eventos.

>PLDxyy<

Onde:

  • x = LED que será manipulado [G, R ou A]
  • yy = Tempo em que o LED ficará aceso em segundos x10 (30 = 3s)

 

Exemplos:

Liga LED verde por 2s

>PLDG20<

Liga LED vermelho por 3s

>PLDR30<

Liga LED âmbar (vermelho+verde) por 4s

>PLDA40<

 

Os LEDs podem ser manipulados também como se fossem saídas físicas do Vircom, com os comandos SXP:

>SSXP8y<

  • y = 1 acende LED verde
  • y = 0 apaga LED verde.

>SSXP9y<

  • y = 1 acende LED vermelho
  • y = 0 apaga LED vermelho

 

¶ Controle de backlight

No comando de criação de tela ou nas ações dos eventos, podemos manipular o backlight do Vircom, determinando o tempo em que o backlight ficará ligado no nível máximo.

Exemplos:

>TSBL90< Liga a luz de fundo no nível máximo por 90s; após esse tempo, a luz de fundo passará para o nível mínimo

>TSBL00< Liga a luz de fundo no nível mínimo instantaneamente

>TSBL9999< Liga a luz de fundo no nível máximo indefinidamente

 

¶ Retorno sonoro

Ação simples para reproduzir um retorno sonoro ao pressionar um botão.

Exemplo reproduzindo um TIC ao pressionar o botão de Menu:

>TSSE0000 CB01++ +- SGN NN {TIC}<

 

¶ Variáveis

 

¶ Exibição de variáveis

Para exibir valores em tela, utiliza-se o comando TVAR nas ações dos eventos, como exemplificado a seguir.

>TSSE0000 CK00++ +- SGN NN {TVAR,48,3,VUX12}<

Onde:

  • 48 = Posição em X
  • 3 = Posição em Y
  • VUX12 = Valor capturado pelo UT12

 

Outra opção é usar o comando TPCM também na ação dos eventos, conforme o exemplo:

>TSSE0000 CK00++ +- SGN NN {TPCM,48,3,QCT10,7,5}<

Onde:

  • 48 = Posição em X
  • 3 = Posição em Y
  • QCT10 = Consulta a ser feita
  • 7 = Caracteres a ignorar no retorno da consulta
  • 5 = Caracteres a exibir

 

 

¶ Variáveis de usuário

O Vircom disponibiliza 32 variáveis do tipo signed int de 32 bits, ou seja, podem assumir valores de -2147483648 a 2147483647.

 

Setando valores:

>TSUS00653<

>TSUS01-86<

 

Consulta:

>TQUS02<

 

Onde:

  •     = Índice da variável (00 a 31)
  •     = Valor (-2147483648 a 2147483647)

 

Para exibir os valores das variáveis de usuário em tela, utiliza-se a referência USRxx dentro da ação TVAR, por exemplo:

>TSSE0000 CK00++ +- SGN NN {TVAR,48,3,USR12}<

¶ Configurações

 

¶ Contraste do display

>TCFG06,xx<

Onde:

  • xx é o valor do contraste, podendo ser de 0 a 63 – padrão: 22

 

¶ Backlight

 

¶ Intensidade do backlight com ignição desligada (dia)

>TCFG07,aabb<

Onde:

  • aa = brilho mínimo [0 a 99] – padrão: 00
  • bb = brilho máximo [0 a 99] – padrão: 40

 

¶ Intensidade do backlight com ignição desligada (noite)

>TCFG09,aabb<

Onde:

  • aa = brilho mínimo [0 a 99] – padrão: 10
  • bb = brilho máximo [0 a 99] – padrão: 40

 

¶ Horário para definir dia/noite (verão)

>TCFG10,aabb<

Onde:

  • aa = horário para início do dia [0 a 23] – padrão: 09
  • bb = horário para início da noite [0 a 23] – padrão: 23

 

¶ Horário para definir dia/noite (inverno)

>TCFG11,aabb<

Onde:

  • aa = horário para início do dia [0 a 23] – padrão: 10
  • bb = horário para início da noite [0 a 23] – padrão: 21

 

¶ Mês para definir verão/inverno

>TCFG12,aabb<

Onde:

  • aa = mês de início do verão – padrão: 11
  • bb = mês de início do inverno [0 a 23] – padrão: 05

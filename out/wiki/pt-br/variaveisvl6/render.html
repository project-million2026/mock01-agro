¶ ? Contadores Numéricos - CTs

Contadores numéricos que permitem armazenar dados, realizar operações matemáticas e registro de dados. 

Variável inteira de 32 bits – 4 bytes (0 – 4.294.967.295).

  • No VL6 podem ser utilizados até 127 CTs, com índices de 0 a 127.
  • No VL8, VL12 e VC7 podem ser utilizados até 192 CTs, com índices de 0 a 95 e de 100 a 195.

⚠️  Não é possível armazenar números negativos dentro de um CT.

 

¶ ⚙️ Configurando CTs

>SCTxx mmmmmmmmmm<

Onde:

xx = Número do contador CT;

mmmmmmmmmm = valor.

 

¶ ? Consultando CTs

Consulta em decimal:

>QCTxx<

Onde:

xx = Número do contador CT;

 

Consulta em hexadecimal:

>QCTxxHEX<

Onde:

xx = Número do contador CT;

 

Consulta faixa de CTs: 

>QCTaaabbb<

Onde:

aaa = índice do CT inicial da consulta;

bbb = índice do CT final da consulta.

 

Consulta faixa de CTs em hexadecimal: 

>QCTaaabbbHEX<

Onde:

aaa = índice do CT inicial da consulta;

bbb = índice do CT final da consulta.

 

Atenção para os retornos das consultas em CTs acima de 100, já que retornam um dígito a mais.

? Exemplos:

>RCT05 00000;ID=3333;#8008;*46<

>RCT132 00000;ID=3333;#8009;*72<

 

¶ ? Limpando CTs

Limpeza de um CT: 

>CCTxx<

Onde:

xx = Número do contador CT;

 

Limpeza de uma sequência de CTs: 

>CCTaaa..bbb<

Onde:

aaa = índice do CT inicial;

bbb = índice do CT final. 

 

¶ ? Operações Matemáticas com CTs

Podem ser realizadas operações de adição, subtração, divisão e multiplicação com CTs dentro das ações de eventos. A precedência das operações é na ordem de escrita (da esquerda para a direita).

 

? Exemplos:

>SED048 TT00++ IN07++ SGN NN {SCT25 VCT08*1000}< 

Multiplica o valor do CT08 por 1000  e salva o resultado no CT25.

 

>SED048 TT00++ IN07++ SGN NN {SCT25 VCT25/VCT08}< 

Divide o valor do CT25 pelo valor do CT08 e salva o resultado no CT25.

 

>SED048 TT00++ IN07++ SGN NN {SCT25 VCT25+VCT08-10}< 

Soma o valor do CT25 com o valor do CT08, subtrai 10 e salva o valor no CT25.

 

¶ Manipulacao de bits HT/QT/BT (FW min: 160B25 - VL8/VL12/VC7)

A partir do firmware 160B25 estao disponiveis novos operadores para manipular partes de um CT (32 bits) com granularidade de 16 bits (HT), 8 bits (QT) ou 1 bit (BT). Util para empacotar varias variaveis em um unico CT ou para extrair flags de bytes recebidos do CAN.

Mapeamento de bits dentro de um CT (32 bits):

  • HT0/HT1 - Half-word (16 bits cada). HT0 = bits 0-15, HT1 = bits 16-31
  • QT0..QT3 - Quarter-word (8 bits cada). QT0 = bits 0-7, QT1 = 8-15, QT2 = 16-23, QT3 = 24-31
  • BT0..BT31 - Bit individual (0 ou 1). BT0 = bit 0, ..., BT31 = bit 31

Exemplo (gravar valor 5 nos bits 16-23 do CT10, preservando o resto):

>SED050 TT00++ IN07++ SGN NN {SCT10 QT2 5}<

Exemplo (ler o bit 3 do CT05 para uma flag):

>SED051 TT00++ IN07++ SGN NN {SSH001 VCT05_BT3}<

Diferente do HT/QT/BT (que exige FW 160B25+), os operadores bit a bit funcionam em qualquer firmware. Mesma precedência das demais operações: esquerda para a direita.

Operador & (E / AND): o bit do resultado é 1 só quando os DOIS bits são 1. Serve para mascarar/filtrar (deixa passar só os bits da máscara).

Operador | (OU / OR): o bit do resultado é 1 quando QUALQUER um dos bits é 1. Serve para ligar/juntar bits.

Exemplos:

>SED048 TT00++ IN07++ SGN NN {SCT88 VCT88|VCT16}<

OU bit a bit entre CT88 e CT16, salva no CT88 (junta os bits ligados dos dois).

>SED048 TT00++ IN07++ SGN NN {SCT88 VCT88&65535}<

E com 65535 (0xFFFF): mantém só os 16 bits baixos e zera o resto (máscara).

>SED048 TT00++ IN07++ SGN NN {SCT88 VCT88&2147483648}<

Mantém só o bit mais alto (bit 31): resultado 2147483648 se ligado, 0 se não. Útil para testar/isolar um bit de flag.

Obs.: a máscara (& valor) é destrutiva - zera os bits fora da máscara. Para testar o bit n, use a máscara 2^n (bit 0 = 1, bit 1 = 2, bit 15 = 32768…). Em FW 160B25+, para acessar um bit/byte preservando o resto, prefira BT/QT/HT.

¶ ? Comparando CTs no VL6

Podemos utilizar as funções de comparação nas ações dos eventos para configurar disparadores que determinam se um CT é maior, menor ou igual a um outro CT.

 

{GEQVCT01VCT02,01}

Compara se o CT01 é igual ao CT02, se for igual dispara o CM01++

 

{GGRVCT01VCT02,02}

Compara se o CT01 é maior que o CT02, se for igual dispara o CM02++

 

{GSLVCT01VCT02,03}

Compara se o CT01 é menor que o CT02, se for igual dispara o CM03++

 

? Exemplo:

>SED000 TT00++ +- SGN NN {GEQVCT01VCT02,01}< // Evento que faz a verificação a cada segundo

>SED001 CM01++ +- TRM AC IGUAL< // Dispara quando a comparação for verdadeira (CT01==CT02)

>SED002 TT00++ +- SGN NN {GGRVCT01VCT02,02}< // Evento que faz a verificação a cada segundo

>SED003 CM02++ +- TRM AC MAIOR< // Dispara quando a comparação for verdadeira (CT01>CT02)

>SED004 TT00++ +- SGN NN {GSLVCT01VCT02,03}< // Evento que faz a verificação a cada segundo

>SED005 CM03++ +- TRM AC MENOR< // Dispara quando a comparação for verdadeira (CT01<CT02)

 

¶ ? Comparando CTs no VL8/VC7/VL12

As mesmas funções de comparação do VL6 valem para o VL8/VC7/VL12 com algumas pequenas mudanças:

 

{GEQVCT01VCT02}

Compara se o CT01 é igual ao CT02. Se sim, executa todas as ações seguintes.

 

{GGRVCT01VCT02}

Compara se o CT01 é maior que o CT02. Se sim, executa todas as ações seguintes.

 

{GSLVCT01VCT02}

Compara se o CT01 é menor que o CT02. Se sim, executa todas as ações seguintes.

 

? Exemplo:

SED000 TT00++ +- SGN NN {GEQVCT01VCT02}{SCT20 +1}
//EVENTO QUE COMPARA TODO SEGUNDO SE O VALOR DO CT01 É IGUAL AO DE CT02, SE SIM INCREMENTA 1 AO CT20

SED001 TT00++ +- SGN NN {GGRVCT01VCT02}{SSH011}{CCT30}
//EVENTO QUE COMPARA TODO SEGUNDO SE O VALOR DO CT01 É MAIOR QUE O DE CT02, SE SIM SOBE A FLAG 01 E LIMPA CT 30

SED002 UT00++ +- SGN NN {GSLVCT01VCT02}{SCT10 40}{SSXP01}
//EVENTO QUE COMPARA QUANDO UT00 DISPARAR SE O VALOR DO CT01 É MENOR QUE O DE CT02, SE SIM, SETA 40 NO CT10 E ACIONA A SAÍDA DIGITAL 0

⚠️ Os disparadores CMxx++ NÃO EXISTEM nos VL8/VC7/VL12. A utilização dos comparadores deve ser da forma demonstrada acima.

 

¶ ? Atribuindo valores de uma consulta em CTs

É possível armazenar parte da resposta de uma consulta (caso seja numérica) em um CT. Para isso criamos um disparador configurável (UT) fazendo o corte desejado daquela consulta, e após isso criamos um evento que atribui o valor deste UT no CT. 

?Exemplo:

>SUT00,QTT,33,3,0,0 //UT que captura a velocidade dada pelo QTT (GPS)
>SED00 TT00++ +- SGN NN {SCT00 VUT00} //Evento que roda todo segundo, atribuindo o valor do UT00 no CT00 armazenando a velocidade por GPS.

 

¶ ⏱️ Contagem de tempo com CT

Utilizando os CTs é possível fazer uma contagem progressiva do tempo, a partir da variável anteriormente definida no CT, utilizamos os comandos:

Inicia a contagem do CT:

>PCTxx<

Onde:

xx = Número do CT (do 0 ao 63)

 

Para a contagem do CT:

>TCTxx<

Onde:

xx = Número do CT (do 0 ao 63)

 

Contagem exclusiva (Desativa a contagem de todos os outros CTs do 0 ao 47):

>XCTxx<

Onde:

xx = Número do CT (do 0 ao 47)

 

¶ ? Flags

 Variáveis binárias (ligado-1/desligado-0).

No VL6 podem ser utilizados até 16 flags, com índices de 00 a 15.

No VL8, VL12 e VC7 podem ser utilizados até 32 flags, com índices de 0 a 31.

A partir do firmware 160B25 (VL8/VL12/VC7) foram adicionadas 128 flags extras, totalizando 160 flags com indices de 00 a 159 (SH00-SH159). Os disparadores das flags 32-159 utilizam o prefixo F1xx a F263 (F100 = SH32, F263 = SH159).

Exemplo de disparador para flag SH40 (FW maior ou igual 160B25):

>SED100 F108++ +- ZG0 V1<

Onde F108 corresponde a SH40 (108 - 68 = 40 quando a flag esta na faixa 32-159, ou diretamente F100+offset).

O comando >CCT96< (FW 160B25+) zera todas as 160 flags de uma vez.

 

Configuração: 

>SSHxxy<

Onde:

xx = Número de flag

y = Valor associado [0..1]

 

Consulta: 

>QSHxx<

Onde:

xx = Número de flag

 

Disparadores:

Flag em 1: SGxx++

Flag em 0: SGxx--

Onde:

xx = Número de flag

 

Consultando flags em hexadecimal:

>QSHH<

?Exemplo:

>RSHH00800002<

(00800002)₁₆ equivale a  (0000 0000 1000 0000 0000 0000 0000 0010)₂ , indicando que as flags 01 e 23 estão altas.

 

¶ TB - Buffer de texto de usos múltiplos

Há um buffer de texto para fins de uso múltiplo, que é um espaço na memória RAM de 255 caracteres (127 p/ VL6).

Este buffer será carregado com cada chamada de comando STB e possui um deslocamento de onde os dados serão carregados.

Desta forma é possível concatenar e adicionar seções de strings de texto a este buffer.

Os textos capturados podem ser respostas de consultas que serão executadas internamente.

Configuração:

>STBxxx consulta1,inicio1,quantidade1 consulta2,inicio2,quantidade2 ...<

Onde:

xxx = Posição dentro do buffer onde as respostas da consulta serão armazenadas. (Se a posição xxx for maior que o comprimento atual do buffer, ela NÃO será armazenada)

consulta1 = Consulta interna de onde os dados serão retirados

inicio1 = É o primeiro caractere que será retirado da resposta da consulta.

quantidade1 = É o número de caracteres que serão retirados das respostas.

Idem para consulta2, inicio2 e quantidade2.

  • Se a consulta começar com PLN, não será alterado e será considerado como texto plano.
  • Se a consulta começar com PLF não será alterado e será tomado como texto plano permitindo caracteres do tipo:
  • /s = Espaço
  • /n = Nova linha

 

Consulta:

>QTBxxx<

Onde:

xxx = Posição de onde o buffer salvo será entregue.

 

? Exemplo:

>STB000,QTT,16,17<

A partir da posição 0 do buffer (primeiro byte), ele carrega a consulta do QTT no Virloc, iniciando no caractere 16, ocupando 17 caracteres (capturando a latitude e longitude). Dessa forma, se você quisesse continuar salvando no buffer de texto sem sobrescrever no que já está salvo, teria que usar o STB018 em diante. Caso queira concatenar mais uma consulta na resposta do QTB000, você pode salvar no STB017. Salvar qualquer consulta no STB016 ou inferior irá sobrescrever o que já foi armazenado.

¶ ? Variáveis Estatísticas - VL8, VL12, VC7

Os VIRLOCS disponibilizam uma série de variáveis estatísticas que são atualizadas com a movimentação do veículo e podem ser consultadas ou zeradas. O evento 200 é reservado e é o responsável por fazer a atualização dessas variáveis.

 

¶ QDM - Distância percorrida

Distância percorrida desde a última vez que a variável foi zerada. É possível realizar a consulta para obter diferentes retornos de acordo com a precisão desejada.

Consulta:

>QDMx<

Onde:

x = modo de consulta, sendo:

  • 1 = reporta em km;
  • 2 = reporta em km com 1 casa decimal;
  • 3 = reporta em km com 2 casas decimais;
  • 4 = reporta em km com 3 casas decimais;
  • 5 = reporta em km com 7 casas inteiras e 3 dígitos decimais.

Respostas:

>RDM2150<

>RDM2150.6<

>RDM2150.65<

>RDM2150.655<

>RDM0002150.655<

 

Zerar:

>CDM<

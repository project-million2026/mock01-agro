¶ Destinos

Os destinos dos eventos definem para onde o reporte será enviado. Tipicamente, os reportes são enviados para um Servidor que tem seu IP configurado na programação, mas também podem ser enviados via Bluetooth ou SMS por exemplo.

? Exemplos:

>SED000 IN07++ +- ZG0 V1<

>SED001 IN07++ +- TRM GP<

>SED002 IN07++ +- BTT GP<

>SED003 IN07++ +- LOG GP<

¶ ⏫ Prioridade dos Destinos

O destino do evento pode ser agregado de um modificador para definir a prioridade da mensagem.

  • _HI: Especifica que o reporte deve ser enviado em prioridade ALTA;
  • _LO: Especifica que o reporte deve ser enviado em prioridade BAIXA;
  • _SL: Especifica que o reporte é enviado em prioridade SUPER BAIXA.

 

? Exemplo: 

GF0_HI : Indica que o reporte vai ser enviado para o IP0 com prioridade alta.

 

¶ ? Destinos Múltiplos

É possível que um evento reporte uma mensagem para até dois destinos utilizando os operadores &&& ou |||.

? Exemplos:

Envia o mesmo reporte para GF0 e GF1 (entrega a mensagem para ambos).

>SED003 IN07++ +- GF0&&&GF1 GP<

 

Envia o mesmo reporte para GF0 ou GF1 (caso seja entregue para GF0, não envia para GF1 e vice versa).

>SED003 IN07++ +- GF0|||GF1 GP<

 

¶ ⚙️ Destinos Condicionais (VL8, VC7, VL12)

Configuração:

>SDNxxtttt00 CCyy DDD EEE<

  • xx = Número do destino condicional [00..07];
  • tttt = Atraso após a condição ELSE;
  • CCyy = Condição necessária para enviar o reporte ao DDD. Este parâmetro tem o mesmo formato que a condição de um evento;
  • DDD = se a condição for verdadeira o reporte é enviado para este destino;
  • EEE = se a condição for falsa, o reporte é enviado para este destino.

 

⚠️ DDD e EEE podem ter DNx como destino, mas não ele próprio.

 

Qualquer destino pode ter uma definição de prioridade. Por exemplo: GF0_HI.

Se uma string for definida com qualquer DNs e não houver condição, o destino BUF poderá ser definido para iniciar a pesquisa de DN0 por uma condição válida. Neste caso, o atraso para o início da busca é definido com o último parâmetro tttt.

 

? Exemplo:

>SED10 IN00-- +- DN0 GP<

>SDN00003000 GR21++&&DA21++ GR0_HI DN1<

>SDN01003000 SQ00++&&DA20++ GL0_LO BUF<

 

Neste caso, o evento 10 é enviado para DN0. Se a condição definida em DN0 for verdadeira (GPRS anexado e protocolo DA habilitado em COM2), o reporte será enviado para GR0 com alta prioridade (HI). Caso contrário, será enviado para DN1.

Se a condição indicada em DN01 for verdadeira (SQ = 1 e protocolo DA está habilitado em COM1), o reporte é enviado para GL0 com baixa prioridade (LO), caso contrário é enviado para o BUF após 3 segundos.

⚠️ O destino LOG não é um destino possível para Destinos Condicionais.

 

Quando um reporte é enviado pelo DNx, a condição é avaliada e o destino correspondente é atribuído.

Se esse destino for de prioridade normal ou alta e não receber resposta (ACK), o pacote retorna ao destino condicional para ser avaliado novamente indefinidamente.

Se esta operação não for desejada, o destino deve ser atribuído _LO de baixa prioridade (dentro do destino condicional).

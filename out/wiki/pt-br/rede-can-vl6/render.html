¶ Configurações CAN

Configurações da rede CAN para o VL6.

 

¶ ? Procedimentos básicos de instalação

Identificação dos fios

CANH → Tensão de aproximadamente 2,6 V;

CANL → Tensão de aproximadamente 2,4 V;

Resistência do barramento de aproximadamente 60 Ohms.

 

 

¶ ▶️ Habilitação da Leitura CAN

>VS19_ENA0250,0<

Onde:

0250 = BAUDRATE da CAN [50, 125, 250, 500…]

0 = Modo, sendo:

  • 0 - Escuta (Silent Mode) - O VIRLOC apenas irá “escutar” os dados que trafegam no barramento;

 

Para desabilitar a leitura CAN: 

>VS19_ENA0<

 

¶ ?️ Debug da CAN

Para ter acesso a uma forma de visualização em tempo real do tráfego dos pacotes no barramento, pode ser utilizado o modo de debug SDB09, que irá abrir uma janela separada no XVM Terminal exibindo essas informações.

Ativar o debug da CAN:

>SDB09<

 

Desativar o debug da CAN:

>SDB00<

 

¶ Observações

  • No VIRLOC 6, o debug da CAN somente irá exibir as PGNs que estão sendo atualmente capturadas.
  • Para qualquer modo de debug exibir os pacotes, lembre-se de ativar a CAN com o comando VS19_ENA explicitado anteriormente.

 

¶ ? Configurações de captura filtros CAN

Cada filtro pode capturar até 4 parâmetros do mesmo PGN (Parameter Group Number), assim formando 4 blocos de captura separados por vírgula:

>VS1901,61444,000,29,01,1,1,000000FF,0,0,1000,02,2,1,000000FF,0,0,1000,03,3,1,000000FF,0,0,1000,04,4,1,000000FF,0,0,1000<

Onde:

Primeira captura:

01 = Número do filtro S19 [00..12];

61444 = PGN;

000 = Source Address [000 = Desativa filtro de Source Address];

29 = Identificador da CAN de 11 ou 29 bits [11, 29];

01 = CT para onde deve copiar ou acumular o valor capturado;

1 = primeiro byte a copiar [1..8];

1 = quantidade de bytes a copiar [1..4];

000000FF = Máscara para aplicar ao valor capturado (mais detalhes abaixo);

0 = Modo [0 = copia valor no CT, 1 = Acumula valor no CT);

0 = Ordem de captura dos bytes da CAN [0= inverter ordem, 1= não inverter ordem];

1000 = Multiplica o valor capturado por 1000 (logo o divide por 1000)

Segunda captura:

02,2,1,FFFFFFFF,0,0,1000

Terceira captura:

03,3,1,FFFFFFFF,0,0,1000

Quarta captura:

04,4,1,FFFFFFFF,0,0,1000

 

Caso seja necessário realizar mais de 4 capturas num mesmo PGN, deve-se utilizar o filtro VS19 seguinte configurado da seguinte forma:

>VS1902,09999,000,99,01,5,1,000000FF,0,0,1000<

Onde:

02 = Número do filtro (deve ser o filtro imediatamente seguinte);

09999 = O campo de PGN deve ser preenchido com 09999;

99 = O campo do Identificador da CAN deve ser preenchido com 99;

  • Os demais parâmetros devem ser preenchidos normalmente conforme detalhado acima, possibilitando até mais 4 capturas.

 

¶ ? Máscara Aplicada aos Bytes Capturados nos Filtros CAN

Para que sejam definidos os bits que devem ser utilizados dentro dos bytes para compor o valor lido, é montada uma máscara. Esta máscara deve acompanhar o número de bytes selecionados nos parâmetros anteriores. 

? Exemplo: 

Pacote lido na CAN: J1939 / PGN:00072 / SA:128 / 1122334455667788

 

 

Para capturar todos os bits dos bytes 3 e 4 sem inversão de bytes (byte 4 mais significativo).

Filtro CAN deve ficar:

>VS1901,00072,000,29,01,3,2,0000FFFF,0,1,01000<

Onde:

3 = primeiro byte a copiar

2 = quantidade de bytes a copiar

1 = modo 1 de captura, sem inverter os bytes

0000FFFF = máscara de bits aplicada

Máscara byte 4 = 0000 0011 (bin) = 03 (hexa)

Máscara byte 3 = 1111 1111 (bin) = FF (hexa)

 

Filtro CAN deve ficar:

>VS1901,00072,000,29,01,3,2,000003FF,0,1,01000<

3 = primeiro byte a copiar

2 = quantidade de bytes a copiar

1 = modo 1 de captura, sem inverter os bytes

000003FF = máscara de bits aplicada 

  • 03 = Máscara do byte 4
  • FF = Máscara do byte 3

 

¶ ? Limpeza dos filtros de captura CAN

A limpeza das configurações efetuadas nos filtros CAN pode ser realizada pelo comando:

 CLNpppppppp_CAN

Onde:

pppppppp = password do equipamento, por padrão de fábrica 00000000 (para o VL6) ou 0000 (para o VL8, VL12 e VC7)

 

¶ ? Operações matemáticas na captura CAN

Para transformar os dados capturados através dos filtros CAN, podem ser aplicadas operações matemáticas (subtração, adição, multiplicação e divisão) e filtros para os valores máximos (High) e mínimos (Low) aceitáveis para serem copiados aos CTs.

>VS19xx_MAT,-40x5,+10/4,L10H50,x100L0H100<

Onde:

- x = Operações 

40 5 = Valores

 +10/4 = Operações da segunda captura

 L10H50 = Operações da terceira captura

 x100L0H100 = Operações da quarta captura

Podendo ser:

  • L = Filtro do menor valor permitido
  • H = Filtro do maior valor permitido
  • x (minúsculo) = Multiplicação
  • / = Divisão
  • + = Adição
  • - = Subtração

 

¶ Observações

  • Os blocos de operações separados por vírgula são correspondentes aos blocos de parâmetros do mesmo PGN (por consequência do mesmo VS19xx). Podem ser definidas até 4 operações matemáticas por bloco de captura (inclui filtros L e H).
  • As operações são executadas de forma sequencial (da esquerda pra direita, sem precedência matemática de operação) a partir do valor capturado definido em VS19xx.
  • São permitidos somente valores inteiros nas operações (Max valor: 268.435.455)
  • Quando um valor é filtrado por ser menor ou maior (operações L ou H), o valor não é copiado no CT definido em VS19xx.

 

¶ ? Contadores de erros

Consulta que possibilita verificar o estado de recepção e transmissão da rede, através da qual podemos verificar os contadores de erro.

Consulta:

>VQ19_ST<

Retorno:

>VR19_ST,000,000,00;ID=8C02;#800D;*6B<

Onde:

000 = REC: contador de erros de recepção;

000 = TEC: Contador de erros de transmição;

0= Código do último erro:

  • 0 - No Error
  • 1 -  Stuff Error
  • 2 - Form Error
  • 3 - Acknowledgment Error
  • 4 - Bit recessive Error
  • 5 - Bit dominant Error
  • 6 - CRC Error

0 = Bitmap de flags de erro:

  • Bit0 - Error warning flag
  • Bit1 - Error passive flag
  • Bit2 - Bus-off flag

 

¶ ? Captura de Pacotes Multiframe

A partir do firmware 9.18 está disponível função "F" no filtro matemático para executar a captura de pacotes multiframe na CAN. Pacotes multiframe são transportados num mesmo número de PGN, porém separado em frames. A função é baseada na utilização dos primeiros 4 bytes como identificador do frame, assim o valor do CT só será atualizado ao capturar um pacote cujo valor dos primeiros 4 bytes coincidam com o especificado.

? Exemplo:

>VS1900,61144,000,29,00,5,2,0000FFFF,0,0,1000< //Captura bytes 5 e 6 no CT00 para PGN 61144

>VS1900_MAT,F287454020< //Só atualiza o CT00 se os primeiros 4 bytes de dados são 0x11223344 (287454020 em decimal)

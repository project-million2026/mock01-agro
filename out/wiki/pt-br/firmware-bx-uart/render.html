¶ Introdução

Este procedimento está definido para atualização do firmware dos módulos BX utilizados nos Virlocs que estão com a versão 1.x, a qual não é possível atualizar pela plataforma Sierra.

 

¶ Procedimento

Antes de iniciar o procedimento, faça o download dos seguintes arquivos mais atualizados no site da Sierra:

BX310x Firmware

BX310x Firmware Upgrade Tool

Nota: A ferramenta de atualização Sierra requer que o Visual C++ Redistributable for Visual Studio 2015 esteja instalado no PC. (Disponível em https://www.microsoft.com/en-ca/download/details.aspx?id=48145.)

 

¶ Remover o módulo

O módulo BX deve ser removido da placa virloc para se realizar o procedimento. Isto se deve ao fato de que durante o processo de atualização é preciso acionar um pino GPIO do módulo e este não está acessível nas placas Virloc/Vircom.

Assim o módulo deve ser retirado da placa VL/VC e montado na placa de desenvolvimento Sierra.

 

¶ Conectar o BX ao Docklight

Usando o simulador de comandos (arquivo Simulador de Comandos BX.ptp) através do Docklight.

Conectar o módulo (através do cabo USB) ao Docklight com baud rate de 115200.

Testar a comunicação enviando o comando AT.

¶ Definir GPIO27 alto

Definir GPIO27 do BX na placa de desenvolvimento em nível alto, em 3,3v. 

Isso pode ser obtido usando um jumper entre o pino GPIO27 e o pino 'PADS'.

 

¶ Reiniciar o BX por comando AT

Reinicialize a placa enviando o comando RESET.

Esta é o exemplo que você deve obter após o RESET.

Observe a última lista com “ÀWSIHÀ”, isso significa que o módulo realmente está em modo bootloader.

 

rst:0x1 (POWERON_RESET),boot:0x13 (SPI_FAST_FLASH_BOOT)
configsip: 0, SPIWP:0xee
clk_drv:0x00,q_drv:0x00,d_drv:0x00,cs0_drv:0x00,hd_drv:0x00,wp_drv:0x00
mode:DIO, clock div:2
load:0x3fff80d0,len:4
load:0x3fff80d4,len:540
load:0x40078000,len:0
load:0x40078000,len:18412
entry 0x40078c2c
ÀWSIHÀ

Feche o Docklight.

Nota: Lembre-se de fechar a porta COM no Docklight.

 

¶ Executar o software pelo Prompt de Comandos

1) Abrir a pasta com o software de Atualização Sierra (baixado em BX310x Firmware Upgrade Tool).

Nota: a pasta já deve estar descompactada (sem .zip).

2) Abrir o prompt de comandos do Windows (CMD).

3) Chegar até o caminho da pasta no seu computador com o comando CD. Exemplo:

Se a minha pasta (C:\BX310x FW Updater v2) estiver no disco C:, terei o caminho C:\BX310x FW Updater v2, assim o comando será: cd C:\BX310x FW Updater v2

4) Executar o comando BX31xxBlTest.exe comXX -v , onde se tem XX se deve substituir pelo número da porta COM utilizada. Exemplo:

Se estou utilizando a porta COM10 no Windows, devo executar o comando: 

BX31xxBlTest.exe com10 -v 

Assim o módulo deverá responder com a sua versão de bootloader.

 

 

Nota: Se você obtiver connection failed -50, tente novamente, isso significa que a porta não está pronta e isso pode ser devido ao Windows não ter concluído o fechamento da porta ou a inicialização da porta.

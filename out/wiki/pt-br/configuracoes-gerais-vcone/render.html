¶ Configurações Gerais

Principais configurações para o VCONE

 

¶ Controle de Energia

Desligando dispositivos

>VSLP0000,xxxx<

Ativando dispositivos

>VSLPFFFF,xxxx<

Onde:

xxxx = Bitmap dos dispositivos para ligar/desligar.

Bitmap dos dispositivos:

  • 0001 = GNSS
  • 0002 = MDM
  • 0004 = RF (Lora)
  • 0008 = Potência MDM
  • 0020 = Fonte de alimentação
  • 0040 = Entrada pullup 0
  • 0080 = Entrada pullup 1
  • 0100 = Eventos.
  • 0800 = Carregador de bateria.
  • 1000 = Driver Ethernet.

 

¶ Configurações de ignição e alimentação

>VSPKVmain,Vign,INign<

Onde:

  • Vmain = Tensão para detectar alimentação presente
  • Vign = Tensão para detectar ignição ligada
  • INign = Entrada para detectar ignição [0, 1, 2]. Se esse parâmetro for maior que 2, detecta ignição pela tensão de alimentação.

Utiliza os mesmos delays das entradas digitais.

 

¶ Configurações do Modem

O VCone possui um slot para SIM Card.

As configurações do SIM Card podem ser realizadas conforme descrito abaixo:

 

¶ APN

>SIS04AT+CGDCONT=1,"IP","generica.claro.com.br"<

>SIS16"claro""claro"<

Onde:

  • generica.claro.com.br = APN do SIM
  • claro = Usuário do SIM
  • claro = Senha do SIM

 

¶ Consulta de Strings do Modem

>QISxx<

Onde:

  • xx = número da String

 

Por exemplo, para consultar APN do SIM card:

>QIS04<

Resposta:

>RIS04AT+CGDCONT=1,"IP","allcom.vivo.com.br";ID=0486;#800C;*0F<

 

 

¶ Configuração do IP de destino

O VCONE tem a possiblidade de configuração de três IPs de destino para as mensagens, Socket 0, Socket 1 e Socket 2 (IP0,  IP1 e IP2).

 

Para consultar os destinos, enviar o comando:

>QIPx<

Onde:

  • x = 0 – IP 0 (Socket 0)
  • x = 1 – IP 1 (Socket 1)
  • x = 2 – IP 2 (Socket 2)

 

Configuração:

>SIPx,aaa.bbb.ccc.ddd,ppppp,m,F<

Onde:

  • x = Número do endereço IP [0, 1 ou 2]
  • aaa...ddd = Endereço IP destino
  • ppppp = Porta de destino
  • m = Tipo de Socket - 0 = UDP
  • F = Socket (IP) alternativo (1 = Socket 1, F = desabilitado)

 

 

¶ Configuração de destino DNS

Para configurar o VIRLOC 6 para envio em um DNS, o IP 0 e o IP1 devem ser preenchidos com 0.0.0.0, ou outro IP determinado opcionalmente, e a porta utilizada nos respectivos DNS conforme o exemplo abaixo:

>SIP0,0.0.0.0,ppppp,0,1<

>SIP1,0.0.0.0,ppppp,0,F<

>SIP2,0.0.0.0,ppppp,0,F<

Onde:

  • ppppp = porta de destino

 

Os DNS podem ser consultados com os comandos abaixo:

>QIS24< Consulta DNS de IP0

>QIS25< Consulta DNS de IP1

>QIS26< Consulta DNS de IP2. 

Configuração:

>SIS24www.virtec.com.ar< Configura DNS IP0

>SIS25< Desabilita DNS IP1

>SIS26www.virtec.com.ar< Configura DNS IP2. 

 

 

¶ Reportes Configuráveis

Consulta da configuração:

>QUCxx<

 

Configuração:

>SUCxx CMD,from,len CMD,from,len CMD,from,len CMD,from,len<

Onde:

  • xx = Índice do reporte configurável [00 a 03];
  • CMD = Comando a executar;
  • from = Posição inicial da resposta do comando;
  • len = Quantidade de caracteres a capturar da resposta do comando.

 

Consulta do reporte (retorna a string final formada pelo UC):

>QUSxx<

 

Diferente dos Virlocs, no VCone o reporte configurável com número de evento é referenciado com a letra U):

>SED002 SG01++ +- TRM Ux<

x = Índice do reporte configurável [U0 a U3].

 

 

¶ Reset do modem

Para reiniciar o modem GPRS, utiliza-se o comando:

>SVP0 2<

 

 

¶ Baudrate da Serial TTL

Por padrão, o baudrate do VCone é de 115200, e retornará a isso sempre que for resetado para as configurações de fábrica.

Para alterar essa configuração, utiliza-se o comando:

>VSRS3,9600<

Onde 9600 é o baudrate a ser utilizado, podendo ser configurado entre 4800 e 115200.

Importante: para utilizar o módulo de GPS, deve-se setar o baudrate para 9600. É recomendado acresentar esse comando logo no final do seu script, ou enviá-lo separadamente após a programação.

 

 

¶ Configuração Ethernet

O VCONE possui uma porta Ethernet, podendo obter conexão diretamente de uma porta LAN de um modem externo ou roteador através de um cabo RJ-45.

A configuração pode ser realizada conforme descrito abaixo:

Consulta: 

>VQET< 

Resposta: 

>VRET192.168.000.023,255.255.255.000,192.168.000.001,DHCP1,MAC3600D2054243,SC100,LS 1,SPD100st200dl0000<

Configuração:

>VSET192.168.000.023,255.255.255.000,192.168.000.001,DHCP1,SPD:10M<

 

Onde: 

  • 192.168.000.023 = Endereço IP local
  • 255.255.255.000= Máscara de subrede
  • 192.168.000.001= Endereço gateway
  • DHCP1= DHCP habilitado [0..1]
  • SPD:10M= Velocidade de conexão LAN (10M ou 100M).
  • MAC3600D2054243= MAC address local
  • SC100 = Estado de conectividade com os sockets 0, 1 e 2 [0 = offline | 1 = online]
  • LS1= Link Status
  • SPD100= Velocidade 100 Mbps
  • st200 = Status (200 = DHCP ready)
  • dl0000= Delay para ligar o driver em unidades de 10 ms.

Se o cabo estiver conectado, cada tentativa de estabelecimento de Link será feita a cada 60 segundos.

O contador dl indica o tempo para a próxima tentativa de conexão, onde 100 representa 1 segundo.

Recomenda-se definir a velocidade para 10M para evitar possível interferência em alguns canais do módulo RF.

 

 

¶ Senha Geral

A senha geral padrão do VCONE é 00000000, podendo ser alterada através do procedimento abaixo.

Defina uma senha geral de 8 dígitos. Deve ser enviada 3 vezes com um indicador de sequência.

>SPEs,aaaaaaaa,bbbbbbbb<

Onde:

  • s = Numero de sequência (1, 2, 3)
  • aaaaaaaa = Senha atual
  • bbbbbbbb = Nova senha

 

 

¶ Trocando ID

O ID é uma cadeia de 4 caracteres alfanuméricos que representa o equipamento e permitirá reconhecê-lo dentro da frota. De fábrica, é configurado um ID correspondente aos quatro últimos caracteres do serial number do equipamento, mas pode ser modificada pelo usuário.

>VSIDpppppppp,DDDD<

  • pppppppp = Senha geral
  • DDDD = Novo ID

Obs.: alterar o ID elimina toda a memória do buffer FIFO/LIFO.

 

 

¶ Reset para a configuração de fábrica

Apagando as configurações:

>CLNpppppppp<

 

Apagando os eventos:

>CLNpppppppp_EVT<

 

Apagando o buffer FIFO/LIFO e LOG:

>GFFEpppppppp<

 

Obs.: pppppppp = password geral (padrão 00000000).

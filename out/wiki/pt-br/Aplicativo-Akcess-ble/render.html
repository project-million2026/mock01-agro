¶ Tela Inicial após a conexão

Após conectar, você verá a tela principal do dispositivo, dividida em abas: SLOT, CONFIGURAÇÃO e DISPOSITIVO

 

 

¶ ⚙️ Configuração dos Slots (Canais de Transmissão)

Após conectar, você estará na aba SLOT. O AS200 possui canais independentes (Slot 1, 2 e 3.), permitindo que ele envie diferentes tipos de informação ao mesmo tempo. 

 

 

Não configurar todos os slots como “no data”, o dispositivo parará de transmitir e o usuário precisará restaurar o dispositivo para padrão de fábrica.

Clique em um Slot para configurá-lo e escolha o Frame Type (Tipo de Quadro):

 

  • Sem info: (Recomendado) Envia os dados dos sensores (movimento, temperatura etc.).
  • iBeacon: Padrão da Apple para localização interna.
  • UID / URL: Padrões do Google (Eddystone) para identificação ou envio de links web.
  • TLM: Informações de diagnóstico (bateria, tempo de atividade).
  • Sem dados: Desativa o slot para economizar bateria.

 

 

¶ ⚙️ Parâmetros de Envio

Dentro do slot, você ajusta como a informação é enviada:

 

  • Intervalo de envio (no lugar de adv): Define a frequência de envio

Exemplo: 1000ms = 1 segundo. Intervalos maiores economizam bateria, mas tornam a detecção um pouco mais lenta.

RSSI@1m: É a força do sinal medida pelo receptor a 1 metro de distância do dispositivo.

RSSI@0m: É a força do sinal medida pelo receptor a 0 metros (encostado) do dispositivo.

 

Como configurar: Este valor deve estar de acordo com testes reais no ambiente de instalação. Geralmente, mantém-se o padrão de fábrica, a menos que seja necessária uma calibração fina para melhorar a precisão da localização.

 

Dica: Potências mais altas (ex: +6dBm) vão mais longe, mas gastam mais bateria. Potências baixas (ex: -20dBm) são para curto alcance.

 

  • Trigger: O Gatilho serve para o AS200 agir de forma inteligente: ele pode ficar "dormindo" para economizar bateria e "acordar" apenas quando algo importante acontecer.

Para configurar, ative a chave Trigger dentro do Slot e siga os 3 passos:
 

1 - A Condição: Escolha o que dispara o alerta:

  • Movimento: Se o objeto se mexer ou parar.
  • Temperatura/Umidade: Se o ambiente ficar muito quente, frio ou úmido.
  • Magnético: Se uma porta ou janela for aberta (requer ímã).

 

2 - Ação Pós-Gatilho (Durante o evento): Defina como o Sensor se comporta enquanto o evento ocorre (ex: porta aberta).

Recomendação: Configure para enviar dados rapidamente (ex: a cada 100ms) para que o sistema receba o aviso imediatamente

 

3 - Ação Pré-Gatilho (Estado normal): Defina como o Sensor se comporta quando está tudo normal (ex: porta fechada).

Recomendação: Configure para enviar dados lentamente (ex: a cada 2 ou 5 segundos) ou até desligar o envio para máxima economia de bateria. 

 

 

Após finalizar as configurações, clique no botão de salvar no topo direito da tela.

 

¶ ⚙️ Aba de Configurações Gerais

Aqui você realiza ajustes globais do equipamento:

 

¶ Configuração do Sensor

Calibre ou ajuste a sensibilidade dos sensores.

 

¶ Acelerômetro

Configura como o sensor de movimento deve se comportar. Isso é crucial para economizar bateria e evitar alarmes falsos.

Opções: 1Hz, 10Hz, 25Hz, 50Hz, 100Hz.

Nota: Frequências mais altas são mais precisas, mas gastam mais bateria.

 

Opções: ±2g, ±4g, ±8g, ±16g.

Uso: Para detectar batidas fortes, use escalas maiores (±16g). Para movimentos cotidianos, ±2g ou ±4g é suficiente.

 

 

¶ Temperatura e Umidade:

Além de visualizar os dados atuais, esta tela permite configurar como o dispositivo grava o histórico climático

 

 

  • Intervalo de Amostragem: Define de quanto em quanto tempo o sensor "acorda" apenas para ler a temperatura. (Aceita valores de 1 a 65535) segundos
  • Sync Time: Sincroniza o relógio interno do sensor com o horário do seu celular
  • Export Dados: Permite baixar o histórico salvo (Excel) ou limpar a memória

 

Quick Switch: Permite configurar atalhos para ligar/desligar o dispositivo via botão físico ou combinações de cliques.

Reiniciar Bateria: Restaura todas as configurações para o padrão de fábrica.

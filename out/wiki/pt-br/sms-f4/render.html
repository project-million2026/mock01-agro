¶ Envio de Comandos via SMS

Para enviar comandos de configuração e consulta para os equipamentos VL08, VL12 ou VC07, é necessário cadastrar previamente os números de telefone autorizados.

  • ✅ Se existirem números cadastrados, o VIRLOC só aceitará SMS vindos desses números.
  • ⚠️ Se não houver nenhum número cadastrado, o VIRLOC aceitará SMS de qualquer origem, respondendo normalmente aos comandos recebidos.

 

¶ ☎️ Cadastrando de números autorizados

Podem ser cadastrados até 4 números diferentes (MA0 a MA3).

Use os comandos abaixo (até 15 dígitos por número):

>SMA0 "numero1" AAA<

>SMA1 "numero2" AAA<

>SMA2 "numero3" AAA<

>SMA3 "numero4" AAA<

 

Para consultar os números cadastrados, basta enviar:

>QMA0<

>QMA1<

>QMA2<

>QMA3<

 

¶ ? Envio de comandos e consultas via SMS

Para enviar comandos/consultas via SMS para VL8, VC7 e VL12, o comando precisa estar no formato completo do protocolo XVM, contendo:

  • ID do equipamento
  • Número da mensagem (#)
  • Checksum (*XX)

Para facilitar, você pode fazer o download da calculadora de checksum aqui que montará o comando completo.

 

No exemplo abaixo, montamos o comando completo do QTT, com ID do equipamento sendo 0000 e número de mensagem 8000 (é importante considerar que as mensagens enviadas ao Virloc devem ser numeradas de # 8000 a #FFFF), retornando:

 >QTT;ID=0000;#8000;*4F<

 

Agora, para enviar efetivamente a mensagem SMS, o comando deve conter (AAA) antes do que for enviado. 

O exemplo do QTT acima ficará dessa forma:

(AAA)>QTT;ID=0000;#8000;*4F<

Assim o VIRLOC responderá o RTT devidamente por SMS.

 

Neste outro exemplo abaixo, foi mandado um QTTrGR0 e um QTTrGR1 via SMS para um VL8, que forçará a mandar a resposta do QTT para o IP0 e para o IP1, respectivamente. Notam-se as respostas ACKs, indicando que ele aceitou os comandos via SMS.

 

¶ ? Modo Transparente (sem protocolo XVM)

A partir do firmware 154B04, existe a opção de configurar o protocolo de SMS em modo transparente, ou seja:
 

Você não precisa mais encapsular o comando no protocolo XVM.
Basta configurar o modo transparente e depois enviar apenas o comando desejado.


? Comando para habilitar o modo transparente:

>SMT03000105020010T120099<

 

Depois disso, os comandos podem ser enviados de forma direta no corpo do SMS (respeitando as regras do equipamento).

 

 

Também é possível concatenar comandos, de forma que o Virloc irá responder a cada comando separadamente.

 

? Também é possível concatenar vários comandos em um mesmo SMS; o Virloc processará cada comando e responderá separadamente.

 

¶ ? Envio de Reports via SMS

O VIRLOC também pode ser configurado para enviar reports automaticamente via SMS.
Os SMS serão enviados para os números cadastrados em MA0, MA1, MA2 e MA3.

 

? Exemplo:

>SMA00 "11986994352" AAA< //Configura número do destino MA0
>STD0100060< //Configura o TD com tempo de disparo a cada 60s
>SED001 TD01++ +- MA0 GP< //Evento que realiza o envio de uma mensagem GP a cada disparo de TD01

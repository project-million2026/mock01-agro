¶ ? Novas Versões de Firmware - VIRLOC 06

Novas versões de firmware tem como finalidade:

  • Adicionar novos recursos;
  • Disponibilizar integração à novos periféricos suportados;
  • Aprimorar o desempenho do equipamento.

 

¶ ? Quando Atualizar o Firmware?

O procedimento de atualização deve ser feito apenas quando detectado que o equipamento já esteja com algum problema em seu firmware atual ou algumas funcionalidades não estejam operando corretamente impactando em seu negócio.

Outra razão para aplicar a atualização é agregar novos recursos e melhorias que a versão disponibiliza.

Importante: Fica a critério do usuário, atualizar ou não o firmware, sendo este, responsável por validar a nova versão junto à sua programação embarcada e a solução aplicada.

 

¶ ✅ Boas Práticas

  • Não tenha pressa, antes de enviar o firmware para um grande número de equipamentos, é aconselhado embarca-lo em um conjunto pequeno e gerenciável de equipamentos a fim de validar seu funcionamento.
  • Os testes de validação podem ser iniciados e executados em bancada, tanto off-line quanto online, esta prática fornece um maior controle nos testes desejados.
  • Para o processo de envio do equipamento para instalação em campo, é aconselhado validar 100% dos equipamentos junto a plataforma. Faça uma checagem se o equipamento se conecta e envia as informações básicas esperadas junto a sua plataforma.

Este simples processo evita surpresas e gastos desnecessários com relação a instalação do seu equipamento.

 

¶ ⚠️ Atenção

Semelhante a todo equipamento eletrônico uma atualização de firmware é um processo crítico e muito importante, e deve ser feito com o máximo de cautela. O download e a atualização do firmware para o equipamento Virloc precisa ser concluído com êxito, para o bom funcionamento do Hardware.

  • Somente atualize o firmware estando o equipamento alimentado diretamente dentro dos limites corretos de trabalho e com uma fonte de alimentação confiável e bateria interna conectada;
  • Não desconecte, desligue nem reinicie o equipamento durante o processo de atualização;
  • Não atualize o firmware estando o equipamento alimentado somente pela bateria de backup (bateria interna);
  • Para upgrade de versões distantes (saltos), consultar a equipe de suporte NEWTEC;
  • Observar com atenção a versão do microcontrolador e a versão da placa do VIRLOC para enviar o firmware correto. A consulta pode ser realizada com o comando QVR, onde a resposta será conforme os exemplos abaixo:
     

>RVR_VL6_FIRM8.04_IAP7_BR59_MDM:2G_F91_SEC:0000<

>RVR_VL6_FIRM9.29_IAP7_BR61_MDM:BG_F91_BT4BSEC:0000_FTEF3013_MIC0256_SBR00<

>RVR_VL6_FIRM10.75_IAP10_BR61_MDM:BG_L48_BT04SEC:0000_FTEF3013_MIC0512_SBR09<

Onde:

8.04 = Versão atual de firmware

BR59 = Versão de placa

F91 = Versão do microcontrolador

¶ Versão 10.01

  • Otimização da leitura de iButton e disparo de IB00++.
  • Reativado modo de debug SDB17 (acelerômetro).
  • Aumento de rotas vetoriais de 4 para 6.
  • Eliminados os parametros VS08 de configuração de detecção de jamming.
  • Adicionado comando para configurar detecção de jamming:

Comando: >SJMmmmmaabbccddeeff<

Consulta: >QJM<

Resposta: >RJMmmmmaabbccddeeff<

Onde:

mmmm = Bitmap de bandas (HEX)

aa = Limites banda 0 (HEX)

bb = Limites banda 1 (HEX)

cc = Limites banda 2 (HEX)

dd = Limites banda 3 (HEX)

ee = Limites banda 4 (HEX)

ff = Limites banda 5 (HEX)

 

  • Agregados parametros de nível e tolerância de ângulo para calibrar acelerômetro:

>GMV_CAL_ABpppp,100,5,2<

Onde:

pppp = Password

100 = Nivel medido para calibrar (min 50, default 100, max 200)

5 = Tolerancia de angulo medido para calibar (min 3, default 5, max 15)

2 = Retardo para validar calibracion (min 1, default 2, max 5),

¶ Versão 8.04

  • Expandida a quantidade de contadores CTs,  de 79 para 128 (índice de 000 a 127).
    • Obs: se utilizado para o dado de velocidade recebida do GPS a configuração anterior previa >VS08,07,99<  agora nas programações atuais devem ser substituídos por >VS08,07,255< pois este parâmetro pode indicar um CT como referência para velocidade e nas versões anteriores não previam o CT99. Por padrão no firmware atual, o valor é 255.
  • Expandida a quantidade de disparadores configuráveis de usuário UTs de 48 para 64 (índice 00 a 63).
  • Agregados novos disparadores UW que disparam com os novos UTs de 48 a 63.

Abaixo temos a lista com todos UTs e seus respectivos disparadores.  

 

 

  • Agregado comando >CLNpppppppp_FULL< para apagar todo o equipamento e restaurar as configurações de fábrica. Onde:

O pppppppp é o password do equipamento com 8 dígitos que por padrão é 00000000.

O CLN FULL tem por funcionalidade:

- Apagar todos os Eventos;

- Apagar todos os CTs;

- Apagar todos os UTs;

- Apagar todos os UCs;

- Apagar todos os IPs, APNs e DNS;

- Apagar o buffer TB;

- Apagar toda memória FIFO/LIFO e de LOG;

- Restaura configurações com Scoring desativado por padrão;

- Restaurar todas as configurações dos VS08, VS16 e IS.

Obs: este comando não é permitido por uma conexão remota, somente por serial TTL e Bluetooth (se disponível).

 

  • Agregado comando >QSHH< para consultar o estado de todas as Flags de uma única vez no formato HEXADECIMAL, assim podendo ser utilizado em um SUC otimizando o tamanho da mensagem. O comando retorna:

>RSHH003F;ID=0000;#8010;*6B< onde 003F é o valor em HEXA das flags de 00 a 05 em 1.

 

  • Agregado comando SSHH para configurar todas as Flags de uma única vez no formato HEXADECIMAL. Exemplos:

- Para configurar todas as flags em 1 podemos com o comando >SSHHFFFF< onde FFFF é o valor em HEXA de 16 bits em 1. 

- Para configurar as flags de 00 a 05 em 1 podemos com o comando >SSHH3F< onde 3F é o valor em HEXA dos 6 primeiros bits em 1.

 

  • Agregado comando para consultar o valor de um CT no formato HEXADECIMAL, assim podendo ser utilizado em um SUC otimizando o tamanho da mensagem. Assim:

>QCTxxHEX<

Onde: xx é o índice do CT a ser consultado e HEX é o identificador do comando para receber a resposta em HEXADECIMAL. Exemplo:

>QCT10HEX;ID=0000;#8017;*0A<

Resposta:

>RCT10HFFFF;ID=0000;#8017;*14<

Onde FFFF é o valor do CT 10 em HEXA (em decimal seria 65535).

 

  • Agregado comando >QCTaaabbb< para consultar o valor de vários CTs de uma única vez, também pode ser utilizado >QCTaaabbbHEX< para retornar os valores no formato HEXADECIMAL, assim podendo ser utilizado em um SUC otimizando o tamanho da mensagem. Sendo:

- aaa índice do CT inicial da consulta;

- bbb índice do CT final da consulta;

 

  • Agregada consulta >QEV< para consulta de data e hora do equipamento.

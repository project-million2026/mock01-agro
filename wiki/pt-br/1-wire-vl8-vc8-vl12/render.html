¶ ?️ Protocolo OneWire – VL08, VC7 e VL12

O protocolo OneWire permite integrar os dispositivos Virloc com:

  • iButtons → identificação de motoristas / usuários
  • Sensores de temperatura → monitoramento térmico embarcado

 

¶ ✅ Requisitos de firmware / hardware

Antes de usar OneWire, verifique:

  • Para todos os Virlocs → firmware ≥ 154B04
  • Para o VL12 → placa a partir da revisão 166_19

Sem isso, o protocolo pode não funcionar corretamente.

 

¶ ⚙️ Habilitação do protocolo OneWire

Nos Virlocs, o OneWire usa 1 entrada digital (IN) e 1 saída digital (OUT) para se comunicar com o sensor/leitor.

 Para definir quais IOs serão usadas, utilize o comando:

>SIO_INTxy_TIM9< 

Onde:

x = Entrada (0 a 4)

y = Saída (0 ou 1)

9 = Base de tempo 1-Wire (padrão 9)

 

¶ ? Casos especiais por modelo

No VL08 é possível usar apenas o fio verde (IN4/OUT0), com a configuração:

>SIO_INT40< //Entrada 4 + Saída 0 (apenas fio verde)

 

No VC07 é possível usar apenas o fio amarelo (OUT1), com a configuração:

>SIO_INT51< //Entrada 5 + Saída 1 (apenas fio amarelo)

 

Outras combinações possíveis:

>SIO_INT00< //Entrada 0 + Saída 0 (fio marrom + verde)

>SIO_INT10< //Entrada 1 + Saída 0 (fio vermelho + verde)

>SIO_INT20< //Entrada 2 + Saída 0 (fio laranja + verde)

>SIO_INT30< //Entrada 3 + Saída 0 (fio azul + verde)

>SIO_INT40< //Entrada 4 + Saída 0 (fio branco + verde)

>SIO_INT01< //Entrada 0 + Saída 1 (fio marrom + amarelo)

>SIO_INT11< //Entrada 1 + Saída 1 (fio vermelho + amarelo)

>SIO_INT21< //Entrada 2 + Saída 1 (fio laranja + amarelo)

>SIO_INT31< //Entrada 3 + Saída 1 (fio azul + amarelo)

>SIO_INT41< //Entrada 4 + Saída 1 (fio branco + amarelo)

 

¶ ⛔ Desabilitar o OneWire

Para desativar o protocolo OneWire:

>SIO_DIS<

 

¶ ? iButton

 

¶ Conexões físicas

 

¶ VL08

Opção 1:

  • Saída: OUT1
  • Entrada: INx (IN0, IN1, IN2 ou IN3)

Opção 2:

  • Somente OUT0 (modo especial descrito com SIO_INT40)

 

¶ VL12

  • Entrada: INx (IN0, IN1, IN2, IN3 ou IN4)
  • Saída: OUTx (OUT0 ou OUT1)
  • Necessário pull-up de 1 kΩ para VOT no barramento OneWire

 

¶ VC07

Opção 1:

  • Entrada: INx (IN0, IN1, IN2, IN3 ou IN4)
  • Saída: OUT0
  • Pull-up de 1 kΩ para VOT

Opção 2:

  • Uso somente da saída OUT1 (modo especial com SIO_INT51)

 

¶ Consultas e disparadores

Para ter acesso aos dados lidos pelo iButton, utilizamos a consulta:

>GIO<

Exemplo de retorno:

>GIO014D44450100003E;ID=0106;#800F;*29<

Onde:

 014D44450100003E = ID da tag lida.

 

¶ ? Apagar o último ID lido

>CIO<

 

¶ ? Disparadores relacionados ao iButton

Cada nova leitura irá alterar o estado dos disparadores:

IO51++ = iButton lido

IO03++ = iButton ou RFID lido

 

¶ ?️ Sensor de Temperatura

 

¶ Conexões físicas

Para utilizar um sensor de temperatura OneWire é necessário ativar a saída de tensão auxiliar disponível no fio da entrada digital 02 (IN2 - FUNCIONAL SOMENTE PARA VL08), para fornecer tensão para o sensor. Nos demais dispositivos (VL12 e VC07) a ligação é feita através do fio VOT. O comando de ativação é o indicado abaixo:

>SSXP21<

¶ ? Leitura de Sensor de Temperatura

Para ler os dados adquiridos pelo sensor de temperatura, utilizamos o:

>GIT<

Exemplo de Resposta:

>GIT0,+0260 R012 C016 TH075 TL070 TM000000800 VAL:1, 00,106E5E0303080006;ID=0285;#8152;*37<

Onde:

+0260 = Leitura de temperatura em °C x10

R012 = Este valor é o resto da leitura. Pode ser usado em conjunto com o valor C para obter leituras mais precisas.

C016 = Este valor é a contagem por grau. Pode ser usado em conjunto com o valor rrr para obter leituras mais precisas. Cálculo: Temperatura = ( TM / 2 ) - 0,25 + [ ( C - R ) / C ]

TH075 = Registrador de temperatura máxima (dado lido do sensor)

TL070 = Registrador de temperatura mínima (dado lido do sensor)

TM000000800 = Tempo desde a última leitura

VAL:1 = 1 leitura válida | 0 leitura inválida

106E5E0303080006= ID do sensor

 

¶ ? Múltiplos periféricos no OneWire

No Virloc 8 é possível fazer a conexão com diversos periféricos (Sensores e Leitores):

Exemplo:

  • 2 sensores de temperatura (2 fios) + 1 iButton;
  • Virloc 8 → IN2 e OUT0 (Pull-up de 1 kΩ entre IN2 e OUT0);
  • GND comum.

Configurações:

>SIO_INT40< = Configura OneWire

>SSXP21< = Ativa VOT na IN2

>GITx< = Consulta dados lidos no barramento

X → Equivale ao índice para consulta

? Observação: o comando >GIO< não é adequado nesse cenário, pois retorna apenas o último ID de iButton do barramento.
Para múltiplos dispositivos, use GITx.

 

Exemplo de resposta:

>GIT0,+0200 R099 C016 TH075 TL070 TM000001412 VAL:0, 00,28D804FE0700007A<

>GIT1,+0220 R099 C016 TH075 TL070 TM000000800 VAL:1, 00,284BFBFD070000D3<

>GIT2,+0000 R000 C000 TH000 TL000 TM000000800 VAL:0, 00,0159E3B601000022<

>GIT3,+0000 R000 C000 TH000 TL000 TM000000800 VAL:0, 00,01826BAE010000CF<

 

  • Sensores sempre ocupam os índices mais baixos da consulta GIT (0 e 1).
  • iButton preenche sempre o próximo índice vazio (neste teste 2, depois 3), sem sobrescrever caso uma nova tag seja aproximada do leitor, logo após a leitura da primeira tag.
  • Se houver espera entre leituras, o mesmo índice pode ser reutilizado para a tag seguinte.
  • Leitura do iButton fica mais lenta quando compartilhando a linha com os sensores, do que somente com o leitor conectado.

 

¶ ? Recomendação prática de uso

  • É viável operar até 4 periféricos no 1-Wire do VL8, ocupando todos os índices da consulta GIT.
  • Para consultar de forma determinística a “última tag” do iButton quando há sensores no barramento, recomendamos criar evento com o comando CIOx  como ação para limpar o primeiro índice após os sensores.

? Exemplo – limpando índice 2 ao ligar ignição

>SED100 IN07++ +- SGN NN {CIO3}<

Aviso : O comando CIOx exige como parâmetro x+1, ou seja, para limpar o índice 2, enviar CIO3.

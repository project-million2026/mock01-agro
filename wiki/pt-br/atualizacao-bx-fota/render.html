¶ Introdução

Para atualizar a versão de firmware do módulo BX3105 via FOTA, é necessário ter:

  • O arquivo BX310x.2.7.3.bin (disponibilizado também no portal de suporte da Sierra).
  • Um servidor HTTP para hospedagem do arquivo, como o HFS.

Importante: Este procedimento só é válido para atualizar módulos que já estejam com firmware superior a 2.5.0.

 

¶ Configurar o servidor HTTP

1. Baixar e instalar em um servidor com IP público o programa HFS HTTP File Server.

 

2. Inicialmente o programa estará configurado para utilizar o IP interno. Para configurar o IP externo, vá até: Menu > IP address > Find external address. Deve ser exibido o pop-up com a mensagem mostrando o IP externo.

 

3. Adicionar o arquivo de firmware com formato .bin em Menu > Add files. O arquivo será exibido na árvore do lado direito da tela.

 

4. Selecionar o arquivo na árvore e copiar o caminho gerado na barra de endereços, ou usar o botão Copy to clipboard.

 

 

¶ Atualização

 

¶ Em bancada

1. Estabelecer conexão com o VL08 ou VC07 via serial TTL.

2. Ativar o modo de debug >SDB14<.

3. Reiniciar o módulo bluetooth com o comando >VSWI_INI< e verificar a versão de firmware atual do módulo. Exemplo: BX310x.2.7.2. Se necessário, parar o debug com >SDB00<.

 

4. Cadastrar uma rede wi-fi para conexão com o comando >VSWIxx,"ssid""password"<.

5. Com o modo de debug ativo, enviar o comando >AT+FOTA=http://3.20.127.27:8080/BX310x.2.7.3.bindDA5< sendo que a parte destacada é a URL gerada pelo servidor HTTP.

6. Observar na janela de debug o progresso do download do firmware.

 

7. Finalizado o download, o módulo deve ativa o firmware e reiniciar automaticamente.

8. Observar a nova versão ser exibida durante a inicialização.

 

¶ Em campo

1. Verificar a versão atual do firmware com a consulta >VQWF<. Exemplo de retorno:

>VRWF4,215,30,5,0,0,99,X2205600951310,cc:93:4a:20:d2:9a,BX310x.2.7.2,000;ID=0061;#80BE;*4B<

2. Garantir que o equipamento possua uma conexão wi-fi. Se necessário, cadastrar uma rede com o comando >VSWIxx,"ssid""password"<.

3. Enviar via servidor o comando >AT+FOTA=http://3.20.127.27:8080/BX310x.2.7.3.bindDA5< sendo que a parte destacada é a URL gerada pelo servidor HTTP.

Obs.: o VIRLOC não enviará uma confirmação para esse comando, então o ideal é testar a comunicação antes.

4. O equipamento continuará operando normalmente durante o download do firmware, e após ativação o módulo BX irá reiniciar, perdendo conectividade wi-fi por alguns segundos.

Obs.: O servidor HFS suporta apenas 1 download por vez.

5. Verificar a nova versão consultando novamente >VQWF<. Exemplo de retorno:

>VRWF4,215,30,5,0,0,99,X2205600951310,cc:93:4a:20:d2:9a,BX310x.2.7.3,000;ID=0061;#80C5;*3B<

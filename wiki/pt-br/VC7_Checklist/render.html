¶ Telas vircom 7 (checklist)

A funcionalidade de Checklist do Vircom 7 permite com que o motorista registre uma inspeção de vários itens do veículo (como cintos, pneus, faróis, etc.) diretamente no display do equipamento.

Ao final, todas as respostas são enviadas em um relatório para o servidor definido pelo usuário. Isso garante um registro digital e confiável de que a vistoria foi realizada, aumentando a segurança e o controle da frota.

A lógica do checklist é construída sobre duas partes principais que trabalham juntas:

  1. As Telas Interativas: São as telas que aparecem para o motorista no display, com as perguntas e as opções de resposta (ex: "SIM" e "NÃO").
  2. O Reporte de Dados: É o "pacote" de informações que o equipamento monta com todas as respostas e envia para a plataforma da empresa.

 

Nessa página vamos ver um exemplo prático de como programar cada etapa do checklist.

 

¶ Passo a Passo: Criando perguntas do Checklist

 

¶ 1. Criando a tela da pergunta

Vamos criar a pergunta: "Cinto está OK?". Podemos usar qualquer uma das 61 telas disponíveis (de 00 a 60). Para este exemplo, usaremos a Tela 19.

 

>TSSB19{TSBL9999}{TSFN2}{TTXT,18,01,CINTO}{TTXT,18,03,ESTA OK?}{TSFN0}{TTXM,85,01,gSIM-mf}{TXM,85,05,gNAO-mf}<

Onde:

  • TSSB19 = Cria a tela de índice 19 [vai de 00 a 60]
  • TSBL9999 = Liga a luz de fundo no nível máximo indefinidamente
  • TSFNx = Define a fonte do Virloc
  • TTXT = Ação de criação de texto
  • 18 = Posição horizontal (coluna em que o texto será inserido)
  • 01 = Posição vertical (linha onde o texto será inserido)
  • CINTO = Texto a ser inserido (sempre em letras maiúsculas)

 

>TSSE1900 CB15++ +- SGN NN {TSSC20}{SCT51 1}< 

  • TSSE1900 = Cria o evento 00 na tela 19
  • CB15++ = Define o evento para acontecer ao pressionar o botão superior
  • +- SGN NN = Sem condicionais e sem envio de reportes
  • TSSC20 = Leva o motorista para a próxima pergunta, que está na Tela 20
  • SCT51 1 = define o CT51 como 1 para registrar a resposta afirmativa.

 

>TSSE1901 CB13++ +- SGN NN {TSSC20}{SCT51 0}<

Onde:

  • TSSE1901 = Cria o evento 01 na tela 19
  • CB13++ = Define o evento para acontecer ao pressionar o botão inferior
  • +- SGN NN = Sem condicionais e sem envio de reportes
  • TSSC20 = Leva o motorista para a próxima pergunta, que está na Tela 20
  • SCT51 0 = define o CT51 como 0 para registrar a resposta negativa.

 

Código na integra:

// 1. Cria a aparência da tela com a pergunta e os botões
>TSSB19{TSBL9999}{TSFN2}{TTXT,18,01,CINTO}{TTXT,18,03,ESTA OK?}{TSFN0}{TTXM,85,01,gSIM-mf}{TXM,85,05,gNAO-mf}<

// 2. Define o que acontece ao apertar "SIM"
>TSSE1900 CB15++ +- SGN NN {TSSC20}{SCT51 1}< //botão CB15++ é definido como a opção SIM, define com o valor 1 o CT51 redireciona para a próxima pergunta (Tela 20)

// 3. Define o que acontece ao apertar "NÃO"
>TSSE1901 CB13++ +- SGN NN {TSSC20}{SCT51 0}<//botão CB13++ definido como a opção NÃO, define com o valor 0 o CT51 redireciona para a próxima pergunta (Tela 20)

Essa lógica se repete para as demais perguntas do checklist

 

¶ 2. Tela de conclusão checklist e envio dos dados para o servidor

Mostra ao usuário que o checklist foi concluído e faz o envio das informações para o servidor:

 

>TSSB26{TSBL9999}{TSFN1}{TIC}{TTXT,25,02, CHECKLIST }{TTXT,25,04,FINALIZADO!}{TSCC0230}{TBEP0040150501500010}{SSH010}< 

  • TSSB26 = Cria a tela de índice 26 [de 00 a 60]
  • TSBL9999 = Liga a luz de fundo no nível máximo indefinidamente
  • TSFNx = Define a fonte do Virloc
  • {TIC} = Ações diversas (opcional)
  • TSCC0230 = contador de tempo decrementais, que disparam ao chegar em 0, define o contador 02 para 30 milisegundos (3 segundos)
  • TBEP = Controla o buzzer interno do Vircom
  • SSH010 = define o valor 0 para a flag 01

 

>SUC04 QTT,4,12 QCT051057_1D,11,99< 

  • SUCxx = criação de reporte configurável
  • QTT = retorna data e hora
  • QCTaaabbb = Consulta uma faixa de CTs (definido do 51 ao 57)

 

>SED098 SG01-- +- GF0 V4 {SSH011}< 

  • SED098 = Evento a ser gerado
  • SG01-- = Quando a flag 01 (SSH01) possuir o valor 0
  • GF0 = Envia para o buffer de saída do servidor definido como IP0
  • V4 = Enviar o report contido no SUC04
  • SSH011 = Define o valor da flag 01 com o valor 1

 

>TSSE2600 CC02++ +- SGN NN {TSSC19}< 

  • TSSE2600 = Cria o evento 00 na tela 26
  • CC02++ =  contador de tempo decrementais que dispara ao chegar em 0
  • TSSC19 = Redireciona para a tela 19 (Inicio do checklist)

 

Código na integra:

// 1. Cria a tela de "Finalizado" e prepara o envio
>TSSB26{TSBL9999}{TSFN1}{TIC}{TTXT,25,02, CHECKLIST }{TTXT,25,04,FINALIZADO!}{TSCC0230}{TBEP0040150501500010}{SSH010}< // Cria a tela 26 e formata o texto para exibir a mensagem de finalização do checklist, ao fim define a flag 01 com o valor 0 que é responsável por fazer o controle de reportes enviados ao servidor

// 2. Define o formato do pacote de dados
>SUC04 QTT,4,12 QCT051057_1D,11,99< //definição do pacote SUC04 para enviar as respostas registradas do ct 51 ao 57 para o servidor

// 3. Cria a regra de envio inteligente
>SED98 SG01-- +- GF0 V4 {SSH011}<  //cria o evento para disparar as respostas salvas do checklist (SUC04) para o servidor quando a flag 01 estiver baixa e depois sobe a flag novamente evitando o loop de envio de reportes

// 4. Define o que fazer após a conclusão
>TSSE2600 CC02++ +- SGN NN {TSSC19}< //redireciona novamente para a tela 19 (inicio do checklist) após 2 segundos
